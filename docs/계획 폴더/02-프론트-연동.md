# 프론트엔드 연동 계획

> NativeWind + 토큰, gluestack 금지, 비즈니스 훅 시그니처 보존(경로/추가만). 디자인은 기존 시스템 위에서.

## 1. API 레이어 (`src/api`)
- `hooks.ts`:
  - `MeResponse`/`ProfileUpdate`에 `jlpt_level_word`, `jlpt_level_kanji`, `push_enabled`, `push_quiz_reminder`, `push_marketing`, `provider` 추가.
  - 출석 월별 훅 `useAttendanceMonth(year, month)` → `GET /api/rewards/attendance/?year=&month=`.
- `notifications.ts`(신규): `useNotifications()`(무한), `useUnreadCount()`, `useMarkRead(id)`, `useMarkAllRead()`, `registerPushToken()`, `unregisterPushToken()`.
- `auth.ts`: `kakaoLogin(accessToken)` → `POST /api/auth/kakao/` (구글과 동일 raw axios, 401 우회).

## 2. 화면별

### A. 설정 — 급수 분리 (SettingsScreen)
- "난이도 설정" 한 줄을 **두 줄**로: "단어 급수"(value=jlpt_level_word) / "한자 급수"(value=jlpt_level_kanji).
- 각각 동일한 급수 선택 바텀시트 재사용(어떤 필드를 수정할지 파라미터화). `useUpdateProfile`로 해당 필드만 PATCH.

### B. 출석 달력 실데이터 (AttendanceScreen)
- `useAttendanceMonth(year, month)`로 교체. `days[].attended`로 정확 표시, `quiz_count`를 셀에 작은 숫자/색농도로.
- 월 이동(이전/다음) 버튼 추가(쿼리 year/month). 추정 캡션 제거.
- 미출석이면 기존 체크인 버튼 유지.

### C. 인앱 알림 (신규 NotificationsScreen + 헤더 배지)
- `AppHeader`에 우측 **종(bell) 아이콘 + 미읽음 배지**(useUnreadCount) 추가. 홈 헤더는 좌:캐시 / 우:[알림][설정] 2버튼.
- `NotificationsScreen`: 목록(읽음/안읽음 구분), 항목 탭 → `data.screen` 있으면 해당 화면 이동 + 읽음 처리, 상단 "모두 읽음".
- 라우트 `Notifications` 등록. 설정에도 "알림" 행 → NotificationsScreen(스텁 제거).

### D. 구매(교환) 내역 UI
- `ExchangeScreen` 하단 또는 설정 "구매 내역" 행 → 내역 리스트(`useExchangeHistory`, 이미 존재). status(requested/issued/failed/refunded) 배지 + 상품명 + 일시.
- 설정 "캐시" 섹션에 "구매 내역" 행 추가(내역=Ledger와 별도).

### E. 푸시 설정 토글 (SettingsScreen)
- "알림 · 계정" 섹션에 토글 행: 전체 푸시(push_enabled), 학습 리마인더(push_quiz_reminder), 마케팅(push_marketing). `useUpdateProfile`로 저장.
- 토큰 등록은 클라 FCM SDK 필요 → **선결조건 충족 후** `registerPushToken()` 호출(앱 시작/로그인 시). 현재는 토글 UI + 백엔드 저장까지.

### F. 카카오 로그인 (LoginScreen)
- 구글 버튼 아래 **카카오 버튼**(노란색, 카카오 가이드 톤). 카카오 SDK로 access token 획득 → `kakaoLogin()` → AuthContext.signIn.
- **카카오 네이티브 SDK/앱키 미설정 시**: 버튼 누르면 "카카오 로그인 준비 중(키 설정 필요)" 안내. (빌드 깨짐 방지 위해 SDK 미설치 상태에선 안내만.)

### G. 문의하기
- 현재 `mailto:` 유지. 수신 메일 주소를 실제 운영 Gmail로 교체(상수/`.env`). 본문 템플릿(앱 버전/기기) 프리필.

## 3. 아이콘
- 추가 필요시 `Icon.tsx`에 보강(이미 bell/lock/mail/calendar 추가됨). 카카오 버튼은 자체 색면 + 텍스트로(아이콘 없이도 가능).

## 4. 헤더 변경 영향
- `AppHeader.right`에 다중 노드(알림+설정) 허용은 이미 `right: ReactNode`라 가능 → 홈에서 두 버튼을 묶어 전달.
