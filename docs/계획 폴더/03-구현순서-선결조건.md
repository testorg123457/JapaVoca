# 구현 순서 & 선결조건

## 1. 구현 순서 (의존성 고려)
1. **A. 급수 분리** — User 필드+마이그레이션(백필) → serializer → learning 필터 분기 → 프론트 설정 2급수. (캐시 무관, 영향 작음)
2. **B. 출석 월별** — 조회 API(모델 그대로) → 프론트 달력 실데이터. (읽기 전용, 안전)
3. **C. 인앱 알림** — notifications 앱+모델+API → 이벤트 생성 훅 연결(on_commit) → 프론트 헤더 배지+화면.
4. **D. 구매 내역 UI** — 백엔드 존재 → 프론트만.
5. **E. 푸시 백엔드 스캐폴드** — PushToken+등록 API+User 푸시 플래그+발송 추상화(log) → 프론트 설정 토글.
6. **F. 카카오 백엔드** — provider/kakao_uid 마이그레이션 + 카카오 로그인 엔드포인트 → 프론트 버튼(키 가드).

> 각 단계 끝에 `makemigrations`/`migrate` + `python manage.py check` + 프론트 `tsc --noEmit`로 검증.

## 2. 선결조건 (사용자/외부 — 자율 구현 불가 부분)

### 2.1 푸시(FCM) 실제 발송
- **필요**: Firebase 프로젝트 + Android 앱 등록 → `google-services.json`(apps/mobile/android/app/) + 서비스계정 키(서버 `.env`의 `FCM_*`/`firebase-admin`).
- **클라**: `@react-native-firebase/app` + `/messaging` 설치 + 네이티브 빌드. 설치 후 `getToken()` → `registerPushToken()` 호출.
- 현재 구현 범위: 백엔드 토큰 저장/설정/발송 추상화(LoggingProvider)까지. 크리덴셜 주입 시 `FcmProvider`만 켜면 됨.

### 2.2 카카오 로그인 클라이언트
- **필요**: 카카오 개발자 앱 생성 → **네이티브 앱키**, Android 키 해시 등록. `@react-native-seoul/kakao-login`(또는 카카오 SDK) 설치 + 네이티브 설정.
- 현재 구현 범위: 백엔드 `/api/auth/kakao/`(access token 검증)까지 + 프론트 버튼(키 미설정 시 안내). SDK/키 설정 후 버튼에서 토큰 획득 연결.

### 2.3 문의 수신 메일
- 운영 Gmail 주소 확정 필요(현재 `support@japavoca.app` placeholder). 확정되면 상수 교체.
