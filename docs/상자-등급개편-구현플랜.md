JapaVoca — 캐시상자 5등급 개편 + 터치 개봉 연출 구현 플랜

> 목표: 상자 등급을 3종(일반/레어/잭팟) → **5종**(일반/레어/에픽/전설/잭팟)으로 확장하고,
> 개봉 연출을 **"등급만큼 터치 → 매 터치마다 한 바퀴 회전 + 색 상승"** 방식으로 교체.
> 작업 규칙: [`CLAUDE.md`](../CLAUDE.md). 현황: [`프로잭트-현황.md`](프로잭트-현황.md).
> ⚠️ 캐시 정합성(원장+잔액)은 **건드리지 않음** — 등급/확률/보상범위와 개봉 UX만 변경.

---

## 1. 배경 / 현재 상태

- 상자 등급은 3종. 등급 가중치는 `learning/services.py:25` `_BOX_GRADE_WEIGHTS`, 보상 범위는 `rewards/services.py:20` `BOX_REWARD_RANGE`.
- 등급은 **정답 시점**에 추첨(`_roll_box_grade`)되어 `CashBox`에 저장, 캐시 금액은 **개봉 시점**에 범위 내 균등 추첨(`open_cash_box`).
- 개봉 화면(`apps/mobile/src/screens/quiz/BoxOpenScreen.tsx`)은 상자를 **1탭**하면 바로 서버 개봉 → 결과 연출. 등급별 색은 결과 표시에만 사용(`normal=회색 / rare=brand / jackpot=amber`).
- `BoxOpen` 라우트 파라미터는 `{ boxIds: number[] }` — **등급 정보 없음**.

## 2. 새 등급 표 (확정)

| 등급 | 키 | 확률 | 가중치(int) | 보상 범위(C) | 평균 | 기여분 | 색 | 터치 수 |
|---|---|---|---|---|---|---|---|---|
| 일반 Normal | `normal` | 70.0% | 140 | 10 ~ 40 | 25 | 17.50 | 상자색(브라운) | 1 |
| 레어 Rare | `rare` | 14.0% | 28 | 40 ~ 90 | 65 | 9.10 | 파랑 | 2 |
| 에픽 Epic | `epic` | 11.5% | 23 | 90 ~ 170 | 130 | 14.95 | 보라 | 3 |
| 전설 Legendary | `legendary` | 4.0% | 8 | 180 ~ 280 | 230 | 9.20 | 노랑 | 4 |
| 잭팟 Jackpot | `jackpot` | 0.5% | 1 | 800 ~ 1,200 | 1,000 | 5.00 | 빨강 | 5 |
| **합계** | | **100%** | **200** | | | **55.75C** | | |

- 가중치 합 200 = 확률×2 (정수, 정확). `random.choices(weights=...)`.
- 기존 `normal/rare/jackpot` 키는 **유지**(rename 아님) → 기존 DB 상자 호환. `epic/legendary` **추가**.
- 보상 범위만 바뀐 등급(normal 5-30→10-40 등)은 **미개봉 상자에만 영향**(개봉 시 범위 적용).

## 3. 터치 개봉 연출 설계

### 3-1. 핵심 규칙
- **터치 수 = 등급 tier + 1** (normal=1 … jackpot=5).
- 박스는 normal 색(브라운)에서 시작. **터치할 때마다**: ① 한 바퀴(360°) 회전, ② 다음 등급 색으로 전환.
- 색 상승은 터치 1..tier 에서 발생, **마지막 (tier+1)번째 터치 = 실제 개봉**(서버 호출 + 보상 공개).
  - 예) 잭팟: 브라운→(탭1)파랑→(탭2)보라→(탭3)노랑→(탭4)빨강→(탭5)개봉.
  - 예) 일반: (탭1)즉시 개봉(색 상승 없음).

### 3-2. 서버 호출 타이밍 — **마지막 터치에서 1회만**
- 프론트가 등급을 **미리 알아야** 터치 수/색 시퀀스를 만들 수 있음 → `BoxOpen` 라우트가 등급을 받도록 변경.
- 중간 터치(1..tier)는 **순수 클라 애니메이션**(서버 호출 없음). 마지막 터치에서 `openBox()` 호출.
- 보상 금액은 여전히 **서버가 확정** — 클라가 아는 건 등급(연출용)뿐, 금액은 개봉 응답으로 받음. 정합성 원칙 유지.
- 광고(`AD_EVERY=3`): 기존처럼 "개봉 N회마다" 유지하되, **마지막(개봉) 터치 직전**에 광고 게이트.

### 3-3. 라우트 파라미터 변경
```
- BoxOpen: { boxIds: number[] }
+ BoxOpen: { boxes: { id: number; grade: BoxGrade }[] }
```
- `HomeScreen`: `boxes.data.map(b => ({ id: b.id, grade: b.grade }))` (이미 `BoxItem={id,grade}`).
- `QuizScreen`: `pendingRef`를 `{id,grade}[]`로 — 정답 응답 `res.box_id` + `res.box_grade` 동시 수집.

### 3-4. 더블탭/락
- 기존 `openLockRef`는 "마지막 개봉 터치"에만 적용. 중간 색-상승 터치는 회전 애니 진행 중 추가 입력 무시(애니 락).

## 4. 색 토큰 (theme/tokens.ts)

기존: brand(민트/vermilion), amber(노랑), danger(빨강), info(파랑) 존재. **보라 + 상자 브라운 신규**.

- primitives 추가: `purple: { 50: '#F3EEFE', 500: '#7C4DFF' }`, `brown: { 50:'#F5EFE8', 500:'#B07B4F' }`(상자색).
- semantic(light/dark) 추가 키: `epic`(purple[500]) / `epic-subtle`, `box`(brown[500]) / `box-subtle`.
- 등급→색 맵(개봉 화면):
  | 등급 | useThemeColors 키 |
  |---|---|
  | normal | `box` (브라운) |
  | rare | `info` (파랑) |
  | epic | `epic` (보라) |
  | legendary | `amber` (노랑) |
  | jackpot | `danger` (빨강) |

## 5. 단계별 작업

### 단계 1 — 백엔드: 등급 추가 + 확률/보상 범위
- `rewards/models.py` `CashBox.Grade`: `EPIC='epic'`, `LEGENDARY='legendary'` 추가. `max_length` 10 유지(legendary=9 OK).
- `rewards/services.py` `BOX_REWARD_RANGE`: 5등급 범위로 교체.
- `learning/services.py` `_BOX_GRADE_WEIGHTS`: 5등급 가중치로 교체.
- 마이그레이션: `CharField choices` 변경은 **DB 스키마 무변경**(no-op 마이그레이션 생성됨). `makemigrations` → `migrate`.

### 단계 2 — 프론트: 타입 + 라우트
- `api/quiz.ts` / `api/hooks.ts` / `api/boxes.ts`: `BoxGrade` 유니온에 `'epic' | 'legendary'` 추가.
- `navigation/types.ts`: `BoxOpen` 파라미터를 `{ boxes: {id,grade}[] }`로.
- `HomeScreen` / `QuizScreen`: 네비게이션 호출부 수정(위 3-3).

### 단계 3 — 프론트: 색 토큰
- `theme/tokens.ts`: purple/brown primitive + epic/box semantic(light·dark) 추가. (tailwind config가 semantic을 자동 참조하므로 className도 사용 가능.)

### 단계 4 — 프론트: BoxOpenScreen 개편
- `GRADE_LABEL`/`GRADE_COLOR`/`GRADE_TIER`(터치 수) 맵 5등급.
- 상태에 `tapsLeft` 추가. 탭 핸들러:
  - `tapsLeft > 1`: 회전(`withTiming 360`) + 색 transition, `tapsLeft -= 1`. (서버 호출 X)
  - `tapsLeft === 1`: (광고 게이트 후) `openBox()` → 결과 연출.
- 회전: `rotate` shared value, 매 탭 `+360`. 색: `backgroundColor`/아이콘 색을 등급 시퀀스로 보간(Animated interpolateColor 또는 단계별 set).
- `RewardReveal`: 5등급 색/라벨. 잭팟·전설은 파티클 연출 유지(전설=노랑, 잭팟=빨강 톤).
- 안내 문구: 터치 수 표시("X번 더 탭!") 또는 진행 점(dot) — tier>0일 때.

### 단계 5 — 검증
- 백엔드: `python manage.py migrate` + 셸로 `_roll_box_grade` 분포 sanity, `open_cash_box` 범위 확인.
- 프론트: 타입 체크(`tsc`), 각 등급 상자 개봉 플로우 수동 확인(에뮬레이터). 색·회전·터치 수.

## 6. 범위 밖
- 등급별 **드롭 연출 차등 강화**(lottie 등) — 후속.
- 등급별 광고 정책 차등 — 이번엔 기존 `AD_EVERY` 유지.
- 기존 DB 상자 보상범위 소급 — 미개봉분은 새 범위 자동 적용, 이미 개봉분은 불변(정상).
