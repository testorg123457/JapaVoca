JapaVoca — 잠금화면 학습 구현 플랜

> 목표: 폰을 켜면(잠금화면) 바로 우리 앱의 **퀴즈가 전면에 뜨고**, 풀거나 **밀어서 해제**로 빠져나간다.
> 작업 규칙: [`CLAUDE.md`](../CLAUDE.md). 현황: [`프로잭트-현황.md`](프로잭트-현황.md).
> ⚠️ 큰 기능 — 네이티브(Android) 의존이 커서 **단계(Phase)로 쪼갠다.** 이번 1차 산출물은 **잠금화면용 퀴즈 화면(UI)** 까지.

---

## 1. 범위 분리 (Phase)

| Phase | 내용 | 비고 |
|---|---|---|
| **A (지금)** | **잠금화면용 퀴즈 화면(`LockQuizScreen`) UI** — 시계/날짜, 퀴즈, 앱열기·상자수 버튼, 밀어서 해제 | RN 화면만. 앱 안에서 라우트로 먼저 검증 |
| B | 네이티브 트리거 — 화면 켜짐(`USER_PRESENT`/부팅) 시 액티비티를 **잠금화면 위로** 띄움 | `android/` Kotlin. `setShowWhenLocked`, `FLAG_KEEP_SCREEN_ON`, foreground 조건 |
| C | 빈도/온오프 설정, 어뷰징·정합성(잠금화면에서도 서버 채점 동일), 배터리/안정성 | 설정 화면 + 정책 |

이 문서의 **세부 설계는 Phase A**(화면)에 집중. B·C는 §6에 개요만.

## 2. Phase A — 화면 설계 (이번 작업)

### 2-1. 결정: 새 화면 `LockQuizScreen` (기존 `QuizScreen`과 분리)
- 잠금화면 전용 크롬(시계·밀어서 해제 등)은 **앱 내부 퀴즈엔 불필요** → 별도 화면이 깔끔하고 기존 인앱 퀴즈 플로우를 안 깬다.
- 퀴즈 **핵심 로직(출제/채점/SRS/상자적립)은 공유**: `api/quiz.ts`(`getNextQuiz`/`submitAnswer`) 그대로 사용. 가능하면 출제·채점 상태관리를 `useQuizSession` 훅으로 추출해 두 화면이 공유.
- ✅ 부수효과: 요청하신 **문제유형 라벨 + 지시문**은 인앱 `QuizScreen`에도 역이식 가능(공통 컴포넌트화).

### 2-2. 레이아웃 (위→아래)
```
┌───────────────────────────────────────┐
│  22:15                    [앱 열기 →]   │  ← 좌상단 시계, 우상단 앱열기
│  6월 26일 목요일           [ 🎁 12 ]    │  ← 그 아래 날짜·요일 / 상자수 버튼
│                                       │
│   ┌─────────────────────────────────┐ │
│   │ 한자                            │ │  ← 카드 좌상단 작은 유형 라벨
│   │ 다음 한자의 뜻으로 옳은 것은?     │ │  ← 지시문
│   │                                 │ │
│   │             水                  │ │  ← 큰 문제(hero)
│   │           (みず)                │ │  ← 읽기(있을 때만)
│   └─────────────────────────────────┘ │
│                                       │
│   ┌───────────┐   ┌───────────┐       │
│   │   물      │   │   불      │       │  ← 4지선다 카드(가로로 살짝 긴 직사각형)
│   └───────────┘   └───────────┘       │     2×2 그리드
│   ┌───────────┐   ┌───────────┐       │
│   │  나무     │   │   쇠      │       │
│   └───────────┘   └───────────┘       │
│                                       │
│         밀어서 해제  ›                  │  ← 하단: 오른쪽으로 밀면 해제
└───────────────────────────────────────┘
```

### 2-3. 요소별 스펙
- **시계/날짜**(좌상단): `new Date()` + `setInterval(…, 1000*30)`(분 단위 갱신, 새 의존성 없음). 시각 `HH:mm`(display 타이포·Bold), 날짜 `M월 D일 EEEE`(caption·secondary).
- **앱 열기 버튼**(우상단): 누르면 잠금 해제(Phase B에선 keyguard 해제 요청) 후 `Main/Home` 진입. Phase A에선 `navigation.navigate('Home')`.
- **상자 수 버튼**(우상단): `useUnopenedBoxes().data.length` 표시(코인/기프트 아이콘 + 숫자, amber). 누르면 `BoxOpen` 진입.
- **유형 라벨**: `item_type`(+가나 트랙)→ `한자 / 단어 / 가나`. 카드 좌상단 micro·brand.
- **지시문**: 유형·`question_type` 조합으로 생성.
  | 유형 | question_type | 지시문 |
  |---|---|---|
  | 한자/단어 | word_to_meaning | 다음 {한자/단어}의 뜻으로 옳은 것은? |
  | 한자/단어 | meaning_to_word | 다음 뜻에 해당하는 {한자/단어}는? |
  | 가나 | (글자→로마자) | 다음 글자의 로마자 표기는? |
- **문제**(중앙): `prompt` hero, `reading` 있으면 아래 작게.
- **4지선다 카드**: `choices`(4개) → **2×2 그리드**, 각 카드 가로로 살짝 긴 직사각형(라운드 lg, 그림자 sm). 탭 → 서버 채점 → 정/오답 색(success/danger) 피드백 → 다음 문제 자동 로드.
  - ⚠️ **카드 크기는 내용 길이와 무관하게 항상 고정**(고정 높이 + grid 균등 너비). 긴 답(`図書館` 등)이 와도 카드가 커지지 않고 텍스트만 1줄 말줄임/축소 처리. RN에선 `numberOfLines={1}` + 고정 height + `flex:1` 균등.
  - (대안: 세로 1열 4행 wide row. 목업은 2×2로 제시 — 피드백 후 택1.)
- **밀어서 해제**(하단): `react-native-gesture-handler` Pan. 손잡이를 오른쪽 임계치(예: 화면폭 35%) 이상 밀면 `onUnlock()` → 화면 종료(Phase A: `goBack`/`Home`). 미만이면 스프링 복귀. "밀어서 해제 ›" 힌트 + 살짝 흐르는 시머.

### 2-4. 재사용 컴포넌트 / 신규
- 재사용: `AppText`, `Card`, `PressableScale`, `Icon`, `Tag`, `useThemeColors`, 토큰.
- 신규: `LockQuizScreen`, `SlideToUnlock`(제스처), (선택) `useQuizSession` 훅, `QuizCard`(유형라벨+지시문+문제) 공통화.

### 2-5. 디자인 톤
- [`디자인-시스템-원칙.md`](디자인-시스템-원칙.md) 준수. 둥글둥글, 포인트색 절제(brand 민트=주, amber=상자/캐시).
- 배경: 잠금화면다운 **은은한 민트 그라데이션 → 화이트**. 퀴즈/카드는 elevated 화이트.
- 이모지는 기능 아이콘으로 쓰지 않음(상자수는 `Icon`).

## 3. Phase A 단계별 작업
1. (선택) `useQuizSession` 훅 추출 — 출제/채점/다음문제 상태. 기존 `QuizScreen`도 점진 이관 가능.
2. `components`에 `SlideToUnlock` 추가(gesture-handler Pan + reanimated).
3. `LockQuizScreen` 구현 — §2 레이아웃. 시계/날짜, 앱열기·상자수, 유형라벨·지시문, 2×2 선택지, 밀어서 해제.
4. 라우트 등록(`MainStack`에 `LockQuiz`) + 임시 진입점(설정 DEBUG 또는 홈 버튼)으로 **앱 안에서 먼저 검증**.
5. 유형 라벨/지시문 헬퍼(`quizCopy.ts`) — 한자/단어/가나 × question_type.

---

## 4. 검증
- 타입(`tsc`) + 에뮬/실기기에서: 시계 갱신, 4지선다 정/오답 피드백, 상자수 반영, 앱열기 이동, 밀어서 해제 임계치/복귀.
- 잠금화면 위 표시(Phase B)는 별도 검증.

## 5. 범위 밖(Phase A)
- 실제 잠금화면 위 표시·부팅/화면켜짐 트리거(=Phase B).
- 빈도/온오프 설정·정책(=Phase C).

---

## 6. Phase B·C 개요 (다음 차)
- **B. 네이티브 트리거**: `BroadcastReceiver`(`ACTION_USER_PRESENT`/`BOOT_COMPLETED`) → 전용 `LockQuizActivity` 기동. `setShowWhenLocked(true)`+`setTurnScreenOn(true)`(API 27+), 키가드 위 표시. RN 화면을 이 액티비티에 싱글톱으로 마운트. 배터리/foreground-service 제약, 제조사(삼성) 백그라운드 정책 대응 필요.
- **C. 정책/안정성**: 채점은 잠금화면에서도 **서버 검증 동일**(클라 신뢰 X). 노출 빈도·방해금지·온오프 설정, 광고는 잠금화면 비표시(정책 리스크) 검토.
