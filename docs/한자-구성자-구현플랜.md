# 한자 구성자 분석 구현 플랜

## 개요

퀴즈 해설 패널(AnswerReveal)에서 한자 클릭 시, 해당 한자의 구성자를 재귀적으로 분해해 트리로 보여주는 기능.
AI 없이 `tbl_content_kanji.components` 컬럼만으로 순수 DB 조회.

---

## DB 현황

| 항목 | 수치 |
|------|------|
| 전체 한자 | 2,205개 |
| components 있음 | 2,146개 (97%) |
| components 없음 | 59개 → 말단 처리 |
| 자기 자신 참조 (말단 표시) | 62개 (예: `一(한 일)`) |
| 파싱 실패 케이스 | 0개 |

**저장 형식**: `言(말씀 언) + 吾(나 오)`
- `+`로 구분, 각 항목은 `한자(뜻)` 형식
- 파싱: `+` split → 각 항목에서 `(` 앞 문자 추출

---

## 재귀 조회 동작 방식

```
語
├── 言(말씀 언)
│   ├── 亖(넉 사)   ← DB 없거나 자기참조 → 말단
│   └── 口(입 구)   ← 자기참조 → 말단
└── 吾(나 오)
    ├── 五(다섯 오) ← 말단
    └── 口(입 구)   ← 자기참조 → 말단
```

**말단 조건** (재귀 중단):
1. `components`가 비어 있음
2. 자기 자신만 참조 (예: `口(입 구)`)
3. DB에 존재하지 않는 문자 (부수 전용 이형자 등)
4. 이미 방문한 노드 (순환 참조 방지)
5. 최대 깊이 초과 (안전장치, 4단계 정도)

## 백엔드 구현

### 엔드포인트

```
GET /api/content/kanji/<character>/components/
```

응답 예시:
```json
{
  "character": "語",
  "meaning": "말씀 어",
  "nodes": {
    "語": {
      "character": "語",
      "meaning_ko": "말씀 어",
      "on_reading": "ゴ",
      "kun_reading": "かた.る",
      "components": ["言", "吾"],
      "is_leaf": false
    },
    "言": {
      "character": "言",
      "meaning_ko": "말씀 언",
      "on_reading": "ゲン",
      "kun_reading": "い.う",
      "components": ["亖", "口"],
      "is_leaf": false
    },
    "口": {
      "character": "口",
      "meaning_ko": "입 구",
      "components": [],
      "is_leaf": true
    }
  },
  "root_components": ["言", "吾"]
}
```

### 서비스 로직 (`content/services.py`)

```
build_component_tree(character):
  1. visited = set()
  2. nodes = {}
  3. _walk(character, visited, nodes, depth=0)
  4. return nodes

_walk(char, visited, nodes, depth):
  if char in visited or depth > 4: return
  visited.add(char)

  kanji = Kanji.objects.filter(character=char).first()
  if not kanji: return  # DB 없는 이형자

  children = parse_components(kanji.components)
  is_leaf = (not children) or (children == [char])

  nodes[char] = {
    'character': char,
    'meaning_ko': kanji.meaning_ko,
    'on_reading': kanji.on_reading,
    'kun_reading': kanji.kun_reading,
    'components': [] if is_leaf else [c for c in children if c != char],
    'is_leaf': is_leaf,
  }

  if not is_leaf:
    for child in children:
      if child != char:
        _walk(child, visited, nodes, depth+1)
```

### 성능
- 최대 depth=4, 분기 보통 2~3개 → 최악 3^4 = 81 쿼리
- `Kanji.objects.filter(character__in=[...])` 로 한 번에 bulk 조회로 최적화 가능
- 응답 캐싱 (`@cache_page` 또는 Django 캐시): 한자는 바뀌지 않으므로 무기한 캐시 가능

## 프론트 구현

### 진입점
퀴즈 해설 패널(`AnswerReveal`)의 한자 표시 영역 → 탭 시 모달 오픈

### 모달 구조

```
[語]  말씀 어  ゴ / かた.る
─────────────────────────
구성자
  └ 言 말씀 언
      └ 亖 (부수)
      └ 口 입 구
  └ 吾 나 오
      └ 五 다섯 오
      └ 口 입 구
```

- 노드 탭 시 → 해당 한자 정보로 재진입 (같은 모달에서 스택 네비게이션)
- DB에 없는 이형자: 글자만 표시, 탭 불가

### 데이터 흐름
1. `AnswerReveal`에서 한자 탭
2. `GET /api/content/kanji/{char}/components/` 호출
3. `nodes` 딕셔너리 + `root_components` 배열로 트리 렌더
4. React Query로 캐싱 (staleTime: Infinity)
