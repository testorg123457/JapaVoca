# 복습 화면(세트 완료 즉시) 구현 계획

> 잠금화면 퀴즈 10문제를 다 풀면, 마지막 문제에서 "다음" 버튼을 누르는 **즉시**
> 복습 화면이 나타난다. 30초 쿨다운은 백그라운드에서 그대로 유지되고,
> 타이머가 끝나 새 세트가 준비되면 복습 화면은 **자동으로 닫히며** 새 문제가 시작된다.
> 복습 화면 디자인은 **Variant C(결과 요약 목록)** — 점수 + 문제 리스트 + 탭 펼침.

**Goal:** 세트 완료 직후 결과를 한눈에 보여준다. 사용자 액션 없이 자동 진입 · 자동 종료.

**핵심 제약:**
- 서버 `SET_COOLDOWN` 30초 **유지**. 백엔드 변경 없음.
- 복습 화면은 마지막 문제 "다음" 버튼 즉시 열린다(별도 버튼 탭 불필요).
- 쿨다운 타이머는 복습 화면 헤더 안에 작게 표시.
- 30초 후 새 세트 로드 완료 → 복습 화면 자동 닫힘 → 새 문제 시작.

## 0. 디자인 원칙 (Variant C: 결과 요약 목록)

이 화면은 **퀴즈 페이지(잠금화면)와 디자인이 확실히 달라야 한다.**

| 구분 | 잠금화면 퀴즈 (기존) | 복습 화면 (신규) |
|---|---|---|
| 톤 | 다크 (`LOCK` 팔레트, `#0C0D10` 배경) | **라이트** (흰 배경, 디자인 시스템) |
| 색 소스 | 화면 내 하드코딩 `LOCK` 객체 | **`theme/tokens.ts` 단일 소스** |
| 성격 | 몰입형 풀스크린 잠금 UI | 정돈된 결과 요약 / 학습 회고 |

- 색·간격·라운드·타이포는 **`tokens.ts`만** 사용. 하드코딩 금지.
- `docs/디자인-시스템-원칙.md` 준수:
  - **글자색 기본 = Ink `#1A1A1A`(`gray[900]`)**, 보조는 그레이. 민트를 글자색으로 칠하지 않는다.
  - **민트(brand)는 한 화면에 2~3곳 이내** — 헤더 면, 주액션, 활성 상태에만.
  - 정답=green(`success #2AC171`), 오답=danger(`red #E8432D`)로 기능색 분리.
  - 캐시/북마크 강조=amber(`#FFCE00`).
  - 카드 남용 금지 — 리스트는 **풀폭 행 + 헤어라인 구분선** 기본, 카드는 펼친 상세에만.
- "AI 디자인 냄새" 회피: 전부 민트로 칠하지 말 것. 중립 그레이를 적극 섞는다.

## 1. 데이터 흐름

### 세트 완료 ~ 복습 화면 ~ 새 세트

```
handleNext() (마지막 문제)
  → setLastReview({ setId, completedAt, entries })   // MMKV 영속화
  → setReviewOpen(true)                               // 즉시 복습 화면 열기
  → loadSet()                                         // 백그라운드 쿨다운 시작
      → 서버: cooldown_until 응답
      → setPhase({ type: 'cooldown', cooldownUntil })
      → [30초 후 재시도]
      → 서버: questions 응답
      → setPhase({ type: 'playing', ... })

useEffect([phase.type]):
  → phase가 'playing'/'noContent'로 바뀌면 setReviewOpen(false) // 자동 닫힘
```

`ReviewEntry`는 `QuizSetQuestion` 전체를 담으므로(`detail`, `choices`, `item_type`, `item_id` 포함)
상세 · 듣기 · 북마크에 필요한 모든 정보가 로컬에 있다. **서버 추가 호출 불필요.**

### QuizReviewModal 렌더 타이밍

```tsx
// 복습 모달은 phase와 독립적으로 reviewOpen 상태가 제어
{reviewOpen && lastReviewData && (

## Global Constraints

- 서버 `SET_COOLDOWN` 은 **30초 유지**. 백엔드 변경 없음.
- 복습 화면 색/간격/라운드/타이포는 **`theme/tokens.ts`에서만** 가져온다. 하드코딩 hex 금지.
- 복습 화면은 **라이트 테마 고정**(잠금화면 다크와 시각적으로 분리). 민트는 화면당 2~3곳 이내.
- 복습 화면은 마지막 문제 완료 즉시 **자동 진입**(별도 "복습하기" 버튼 없음).
- 쿨다운 종료 시 복습 화면 **자동 닫힘** → 새 세트 자동 시작.
- 정답 판정값은 이미 서버에서 온 `is_correct`(온라인) 또는 로컬 채점(오프라인)을 **그대로** 저장만 한다. 재판정 안 함.
- 듣기(TTS)는 `react-native-tts`, `ja-JP`, **오프라인 동작**.

const REVIEW_KEY = 'quiz.lastReview';

export type ReviewEntry = {
  question: QuizSetQuestion;
  selectedIndex: number;
  isCorrect: boolean;

export type ReviewData = {
  setId: number;
  completedAt: string;   // ISO
  entries: ReviewEntry[];

export function getLastReview(): ReviewData | null {
  const raw = storage.getString(REVIEW_KEY);
  if (!raw) { return null; }
  try { return JSON.parse(raw) as ReviewData; } catch { return null; }
}

export function setLastReview(data: ReviewData): void {
  storage.set(REVIEW_KEY, JSON.stringify(data));
}

export function clearLastReview(): void {
  storage.remove(REVIEW_KEY);
}
```

복습 화면과 잠금화면이 같은 로직을 공유하되 **버튼 UI는 각자 테마로** 그린다.

- [ ] `src/lib/useSpeak.ts` 신규:

```ts
import { useCallback, useEffect, useRef, useState } from 'react';
import Tts from 'react-native-tts';

// 앱 기동 1회 — ja-JP
Tts.setDefaultLanguage('ja-JP');

/** '・'로 구분된 복수 발음을 순차 재생. 토글/취소/언마운트 정리 포함. */
export function useSpeak(text: string) {
  const [speaking, setSpeaking] = useState(false);
  const cancelRef = useRef(false);
  const subRef = useRef<{ remove: () => void } | null>(null);

  useEffect(() => () => {
    cancelRef.current = true;
    subRef.current?.remove();
    subRef.current = null;
    Tts.stop();
  }, []);

  const toggle = useCallback(() => {
    if (speaking) {
      cancelRef.current = true;
      subRef.current?.remove();
      subRef.current = null;
      Tts.stop();
      setSpeaking(false);
      return;
    }
    const readings = text.split('・').map(r => r.trim()).filter(Boolean);
    if (readings.length === 0) { return; }
    cancelRef.current = false;
    setSpeaking(true);
    const playNext = (i: number) => {
      if (cancelRef.current || i >= readings.length) { setSpeaking(false); return; }
      Tts.speak(readings[i]);
      let sub: { remove: () => void } | null = null;
      const onFinish = () => {
        sub?.remove();
        subRef.current = null;
        if (i + 1 < readings.length) { setTimeout(() => playNext(i + 1), 600); }
        else { setSpeaking(false); }
      };
      sub = Tts.addEventListener('tts-finish', onFinish) as unknown as { remove: () => void };
      subRef.current = sub;
    };
    playNext(0);
  }, [text, speaking]);

  return { speaking, toggle };
}
```

  2. 헤더: `n/10 정답` + `새 세트까지 N초` 타이머 카운트다운.
  3. 리스트: 정답=green 체크, 오답=red x. 행 탭 → 상세 펼침(내 답 red / 정답 green / 읽기).
  4. 듣기 버튼 → 일본어 발음(와이파이 꺼도 동작).
  5. 북마크 토글 → amber 채워짐.
  6. 30초 후 → 복습 화면 **자동 닫힘** → 새 세트 문제 시작.
  7. ✕ 버튼으로 수동 닫기 → 쿨다운 화면 노출(타이머만). 타이머 끝나면 새 세트 자동 시작.
  8. 앱 강제종료 후 재실행 → 쿨다운 중이면 복습 데이터 MMKV에 남아있음(영속 확인).

## 4. 알려진 한계 / 추후

- **북마크 초기 상태 미조회:** 이미 북마크된 항목도 빈 상태로 시작. 서버 조회는 추후.
- **구성 자세히 보기 제외:** 다크 `ComponentTreeModal` 의존이라 라이트 복습 화면엔 미포함. 필요 시 라이트 버전 별도.
- **오프라인 완료 후 복습:** 오프라인에선 쿨다운 응답이 없어 `phase.type === 'cooldown'`이 안 뜰 수 있음. 복습 화면은 열리지만 쿨다운 타이머가 없으므로 자동 닫힘 트리거도 없음 — 수동 닫기로 대체. 추후 검토.
- **세트가 10문항 미만이면** 점수는 `정답수 / 실제문항수`로 표기.
