## Phase 1 — 완료 (2026-06-25)

**한 일**
- **브랜드 컬러 → 포레스트 민트** (`tokens.ts`): brand 램프를 역할 정렬로 교체
  (semantic `brand`=#1F9660 Primary / `brand-active`=#0F7048 Pressed / `brand-strong`=#084D32).
  danger는 별도 `red` 프리미티브로 분리(빨강 유지), coral은 민트 흡수, gradients.brand 민트화.
  `tailwind.config.ts`는 `vermilion` 객체를 펼쳐 쓰므로 자동 동기화.
- **버밀리온 헤더**: semantic 토큰 `header`(#E8432D)/`on-header`(흰색) 추가 + 공용 `AppHeader` 컴포넌트
  (상태바까지 버밀리온, top safe-area inset 소유, back/title/left/right 슬롯).
- **단일 스크롤 골격**: BottomTab 이미 제거됨. Home은 헤더(좌:캐시/우:설정) + 상자 섹션 + 퀴즈 CTA +
  가나/출석/교환 진입 리스트 + 하단 앵커 배너로 재구성.
- **새 화면 스텁 + 라우트 등록**: `KanaScreen`/`AttendanceScreen`/`SettingsScreen`(재작성)/`ExchangeScreen`
  — 각 AppHeader + 자리표시. MainStack에 Home/Quiz/BoxOpen/Kana/Attendance/Settings/Exchange 등록, types 갱신.
- Icon에 `arrow-left`/`calendar`/`bell`/`lock`/`mail` 추가.

**메모**
- 이전 세션의 `HomeWalletSection`/`HomeSettingsSection`은 이제 미사용 — Phase 4/6에서 교환·설정 로직 참조용으로 보존.
- 구 `WalletScreen`은 여전히 deprecated(참조용). 구 `SettingsScreen`은 새 스텁으로 재작성됨.
- 검증: `tsc --noEmit` 0 에러, eslint 0 에러(inline-style 경고만). JS 변경이라 Metro fast-refresh로 반영.

## Phase 2 — 완료 (2026-06-25)

**한 일** (상당 부분은 Phase 1의 홈 재구성에서 이미 충족 → 검토 + 폴리시)
- 홈 세로 순서 확정: 헤더(좌 캐시/우 설정) → **상자 섹션(elevated hero, 가장 큼/중앙)** → 퀴즈 시작 CTA(카드) →
  가나/출석/교환 진입(ListSection 행 + 구분선 + 화살표).
- 카드 정리 기준 적용: 카드 O = 상자 섹션·퀴즈 CTA(누르는 액션 단위) / 카드 X = 가나·출석·교환 진입(가벼운 행).
  카드 안 카드 없음. 캐시 갯수는 카드가 아닌 헤더 텍스트 계층.
- 출석 행은 한 줄 요약(스트릭/유도 문구) + chevron.
- 배경 warm white(bg-secondary), 버튼 민트 기반, 위계는 크기 + elevation으로.

**메모**: 검증 tsc/eslint 0 에러. 상자 섹션 탭 개봉은 Phase 3.

## Phase 3 — 완료 (2026-06-25)

**한 일**
- 홈 상자 섹션을 탭 가능하게 연결: `boxCount > 0`이면 전체 미개봉 상자 id로 `BoxOpen` 진입,
  `0`이면 dim(opacity 0.55) + 비활성 + "퀴즈 풀고 상자 받기" 유도.
- 개봉 플로우는 기존 `BoxOpenScreen` 재활용 — 보상형 광고(3개당 1회) + 등급별 캐시 연출 +
  `wallet`/`boxes`/`daily` invalidate. 홈이 modal 뒤에서 active observer라 invalidate 시 즉시 refetch →
  복귀 시 헤더 캐시·상자 개수 갱신.
- 연타 방지: 홈 `navLockRef`(중복 push 차단, 포커스 복귀 시 해제) + BoxOpenScreen의 openLock + 서버 409 멱등.

**메모**: 코드는 검증된 기존 개봉 플로우 재활용(tsc/eslint 0 에러). **E2E 1회 실기검증은 폰+백엔드 가동 후 필요**.

## Phase 4 — 완료 (2026-06-25)

**서버 조사 결과**: `/api/quiz/next/?mode=`는 **word/kanji만** 지원(`learning.ItemType`에 kana 없음, 그 외 400).
`content.Word.WordType.KANA`는 단어 분류일 뿐 퀴즈 모드 아님 → **가나 퀴즈는 서버 미지원**.
→ 프롬프트 지시대로 **프론트 로컬 데이터 + 로컬 퀴즈**로 구현(백엔드 무수정).

**한 일**
- `data/kana.ts`: 히라/가타 + 로마자 데이터(청음 오십음 / 탁음·반탁음 / 요음) + flat 퀴즈 풀.
- `KanaScreen`: 스크립트 토글(히라/가타) + 가나표(그리드 셀 + 얇은 구분선, 카드 X) + 셀 탭 선택 강조(민트) +
  "가나 퀴즈 시작" 버튼.
- `KanaQuiz`(모달): 로컬 4지선다 10문제, 글자→발음 / 발음→글자 랜덤, 정/오답 색 피드백 + 결과 요약.
  순수 연습(캐시/원장 무관).

**메모**: tsc/eslint 0 에러.

## Phase 5 — 완료 (2026-06-25)

**서버 조사 결과**: rewards 엔드포인트는 `attendance/today`(GET/POST), `daily/today`만. **월별/이력 조회 API 없음.**
→ 프롬프트 지시대로 **백엔드 추가 없이** 가용 데이터로 구현.

**한 일**
- `AttendanceScreen`: 요약(연속 출석 streak hero + 오늘 출석 여부 태그) + **현재 월 달력 자체 구현**(외부 라이브러리 X) +
  미출석 시 출석 체크 버튼(useAttendance).
- 달력 출석 표시는 **streak 기반 추정**(마지막 출석일부터 역산). 일별 퀴즈수는 이력 API 부재로 생략.
  화면 하단에 "추정 표시" 캡션 명시.

**⚠️ 한계/후속**: 정확한 월별 출석/일별 퀴즈수를 보여주려면 서버에 월별 조회 엔드포인트
(예: `GET /api/rewards/attendance/?year=&month=`)가 필요. 현재는 streak 추정. **임의 추가하지 않음.**
**메모**: tsc/eslint 0 에러.
