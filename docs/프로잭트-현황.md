# JapaVoca — 프로젝트 현황 브리핑 (새 세션 Claude용)
 
> 이 문서는 **현재 구현 상태**를 빠르게 파악하기 위한 것이다. 원본 기획은 [`첫-계획서.md`](첫-계획서.md), 작업 규칙은 루트 [`CLAUDE.md`](../CLAUDE.md), 로컬 실행/트러블슈팅은 [`로컬-실행-가이드.md`](로컬-실행-가이드.md) 참고.
> 계획서는 "초안"이고, 구현하면서 바뀐 부분이 있다 → **§7 계획서 대비 변경/미구현**을 먼저 보면 차이를 빨리 안다.
 
---
 
## 1. 한 줄 요약
 
일본어(단어·한자) 학습 + 캐시 리워드 안드로이드 앱. **퀴즈 정답 → 캐시상자 획득 → (광고 보고) 개봉 → 캐시 적립 → 기프티콘 교환(광고)** 이 핵심 루프. 수익 모델은 캐시워크형(AdMob 보상형 광고). 캐시는 **자산**이라 정합성(원장+잔액 트랜잭션)이 최우선 원칙.
 
- 서비스명: 일본어 한자 보카 / JapaVoca / 패키지 `com.japavoca.app`
- 1차 플랫폼: **Android 우선** (iOS 3차)

## 2. 모노레포 구조 & 스택
 
```
JapaVoca/
├── apps/
│   ├── mobile/   # React Native 0.86 (TS, New Architecture). android/ 에 네이티브(Kotlin)
│   └── server/   # Django 6 + DRF (자체 .venv), PostgreSQL
└── docs/
```
- JS workspace 도구(Nx/Turbo/yarn workspaces) **안 씀**. mobile/server는 같은 레포의 독립 폴더.
- 백엔드는 Python venv로 독립.
 
| 레이어 | 채택 |
|---|---|
| 클라이언트 | React Native 0.86 (TS), New Architecture(Fabric/Bridgeless), Hermes |
| UI | **NativeWind(Tailwind) + 자체 토큰** (⚠️ gluestack-ui는 제거됨 — §7) |
| 상태/데이터 | React Query(서버 상태) + React Context(auth) + MMKV(토큰 영속) |
| 애니메이션 | reanimated(+worklets), gesture-handler, lottie |
| 광고 | react-native-google-mobile-ads (AdMob 보상형) |
| 인증 | 구글 소셜 로그인(@react-native-google-signin) + JWT |
| 백엔드 | Django + DRF, SimpleJWT, 커스텀 User(google_uid) |
| DB | PostgreSQL. 테이블명은 전부 `tbl_<app>_<model>` 규칙 |

## 3. 백엔드 아키텍처 (apps/server)
 
루트 라우팅(`config/urls.py`): `/api/auth/` `/api/quiz/` `/api/rewards/` `/api/exchange/`.
 
### 앱별 책임
| 앱 | 책임 | 주요 모델 |
|---|---|---|
| `accounts` | 구글 로그인·JWT 발급, 프로필 | `User`(google_uid 기반 커스텀, `AUTH_USER_MODEL`) |
| `content` | 학습 콘텐츠 (URL 없음 — 퀴즈 통해서만 노출) | `Kanji`, `Word`, `WordMeaning` |
| `learning` | 퀴즈 출제(SRS+오답생성)·채점(SM-2) | `SrsState`, `QuizLog` |
| `rewards` | 캐시(지갑/원장)·상자·출석·데일리 | `Wallet`, `Ledger`, `CashBox`, `Attendance`, `Daily` |
| `exchange` | 기프티콘 교환·광고 SSV 검증 | `GiftExchange`, `AdRewardLog` |
 
### 캐시 정합성 핵심 (⚠️ 절대 원칙 — CLAUDE.md)
- 모든 캐시 변동 = **Wallet 잔액 갱신 + Ledger(append-only) 기록을 하나의 `@transaction.atomic`**.
- `wallet.balance == (ledger earn합 − use합)` 항상 일치. (`rewards.services.wallet_is_consistent()` 로 감사)
- 차감은 행 잠금(`_lock_wallet`) + 조건부로 음수/이중차감 방지. `Ledger`는 수정·삭제 금지, user FK는 `PROTECT`.
- 금액은 전부 정수 캐시(BigInteger), 소수점 없음.
- `rewards.services`: `earn()`, `use()`, `open_cash_box()`, `check_in()`, `get_today_*()`.
 
### 엔드포인트 맵 (현재)
| 메서드·경로 | 설명 |
|---|---|
| `POST /api/auth/google/` | 구글 ID 토큰 → JWT |
| `POST /api/auth/dev-login/` | **DEBUG 전용** 테스트 유저 로그인(OAuth 우회). DEBUG=False면 404 |
| `POST /api/auth/token/refresh/` · `GET/PATCH /api/auth/me/` | 토큰 갱신 / 프로필 |
| `GET /api/quiz/next/?mode=word\|kanji` | 다음 문제 1개(서명된 question_token 포함) |
| `POST /api/quiz/answer/` | 서버 채점 + SRS 갱신 + QuizLog + Daily + 정답 시 상자 생성. **token_hash로 재채점(더블탭) 멱등 차단** |
| `GET /api/rewards/wallet/` · `ledger/` · `boxes/` | 잔액 / 내역 / 미개봉 상자 |
| `POST /api/rewards/boxes/<id>/open/` | 상자 개봉(보상 확정 + earn 원자적) |
| `GET /api/rewards/attendance/today/` · `daily/today/` | 출석 현황·체크인 / 오늘 학습현황 |
| `GET /api/exchange/products/` · `POST request/` · `GET history/` | 상품목록 / 교환요청 / 교환내역 |
| `GET /api/exchange/admob/ssv/` | AdMob SSV 콜백 검증 |
 
### SRS / 채점 (`learning.services`)
- 출제: SRS 기한 도래 항목 우선 → 없으면 미학습 신규(가능하면 유저 급수). 오답은 같은 종류 다른 항목에서 추출.
- 문제마다 **서명된 question_token** 발급 → 채점 시 위조/만료 검증(`InvalidQuestionToken`).
- 채점: 간이 **SM-2**(ease/interval/repetitions/due_at) 갱신, 정답 시 상자 생성(일일 상한 체크).
- `QuizLog`는 **7일 보관**(배치 삭제 예정, `learning/tasks.py`), 장기 통계는 `rewards.Daily` 집계로 보존.

## 4. 프론트 아키텍처 (apps/mobile/src)
 
### 내비게이션 (`navigation/`)
```
RootNavigator (AuthContext.isLoggedIn 으로 분기)
├── Auth (비로그인)   → Login
└── Main (로그인)
    ├── BottomTab     → Home / Wallet / Settings
    ├── Quiz   (push, 풀스크린)
    └── BoxOpen(modal, { boxIds })
```
- 로그인 상태는 `store/AuthContext` + `store/auth.ts`(MMKV, key `auth.accessToken/refreshToken`)가 단일 진실원. 토큰 유무로 스택 전환.
 
### API 레이어 (`api/`)
- `client.ts`: 공용 axios. access 토큰 자동 주입, **401 → refresh 1회 재시도**, refresh 실패 시 `forceSignOut()`. 동시 401은 단일 refreshPromise로 묶음.
- `auth.ts`: 로그인은 raw axios(401 인터셉터 우회). `googleLogin()`, `devLogin()`.
- `hooks.ts`/`quiz.ts`/`boxes.ts`/`exchange.ts`: React Query 훅. `queryClient.ts`.
- AdMob 보상형: `hooks/useRewardedAd.ts`.
 
### 디자인 (`theme/tokens.ts` 단일 소스 + NativeWind)
- 색·간격·라운드·그림자·타이포를 토큰에서만 정의(하드코딩 금지). 톤은 **둥글둥글**, 포인트 컬러 절제, 폰트 Pretendard.
- 공용 컴포넌트 `components/`: `AppText`, `Button`, `Card`, `CashBadge`. (`StyleGuideScreen`로 토큰 확인 가능)

## 5. 화면별 구현 상태
 
| 화면 | 상태 | 비고 |
|---|---|---|
| 로그인 | ✅ 동작확인 | 구글 로그인 + DEBUG "개발용 로그인" 버튼 |
| 홈 | ✅ 동작확인 | 인사+캐시뱃지, 출석체크 카드(스트릭), 오늘 학습현황, 퀴즈 시작, 상자 인벤토리, AdMob 배너 |
| 퀴즈 | ✅ 동작확인 | 한자/단어 4지선다, 서버 채점, 진행/정답수, 다음 문제 |
| 상자 개봉 | 🟡 구현됨(엔드투엔드 미검증) | `BoxOpenScreen` + `useRewardedAd` |
| 지갑 | 🟡 구현됨(미검증) | `WalletScreen`(잔액/내역 + 교환 UI 통합 추정) |
| 설정 | 🟡 구현됨(미검증) | `SettingsScreen` |
| 온보딩(급수 선택) | ❌ 미구현 | §7 |
| 모드 선택(단어/한자) | ❌ 별도 화면 없음 | 홈 → 바로 Quiz(현재 mode 파라미터 미전달) |
 
(✅ = 에뮬레이터에서 직접 확인, 🟡 = 코드 있음/플로우 미검증, ❌ = 없음)

## 6. 핵심 루프 현 구현 정리
로그인(✅) → 홈(✅) → 퀴즈 정답 → 상자 생성(서버 ✅) → 상자 개봉+광고(🟡) → 캐시 적립(서버 earn ✅) → 지갑(🟡) → 기프티콘 교환+광고(서버 로직 ✅ / 발급사는 Mock) → SSV 검증(✅ 서명검증 구현).
즉 **서버 비즈니스 로직은 거의 다 구현**, 프론트는 로그인/홈/퀴즈까지 검증됨, 그 이후 화면은 코드만 있고 실제 플로우 검증 전.

## 7. 계획서(첫-계획서.md) 대비 변경·미구현 ⚠️ (가장 중요)
 
**바뀐 결정**
- **gluestack-ui 제거**: 계획서는 "gluestack-ui + NativeWind"였으나, gluestack v1이 웹 react-dom을 끌어와 RN 번들이 깨져서 **제거**. 현재는 NativeWind + 자체 토큰 + 직접 만든 공용 컴포넌트.
- DB 테이블명에 `tbl_` 접두사 규칙 적용(계획서엔 없던 컨벤션).
- `content` 앱에 별도 API 없음 — 콘텐츠는 퀴즈(`/api/quiz/`)를 통해서만 나감.
- **DEBUG 전용 dev 로그인** 추가(계획서엔 없음): 구글 OAuth 키 없이 테스트하려고 도입. `GOOGLE_WEB_CLIENT_ID` 비어 있으면 실제 구글 로그인은 안 됨.
 
**아직 미구현 / 부분 구현 (MVP 갭)**
- **온보딩(JLPT 급수 선택) 화면 없음** — `User.selected_jlpt_level` 필드는 있으나 선택 UI 미구현.
- **단어/한자 모드 분리 진입점 없음** — 홈에서 mode 없이 Quiz 진입(서버는 `?mode=` 지원하므로 프론트만 붙이면 됨).
- **기프티콘 발급사 = Mock** (`exchange/providers.py`의 `issue_gifticon`). 실제 B2B API 미연동. 상품은 `exchange/products.py` 코드 상수(아메리카노/편의점권 등 mock).
- **데이터 정제 파이프라인**: 퀴즈가 실제 단어로 동작은 하나, `Word.pos`(품사)·`Word.jlpt_level`(AI 태깅)·뜻 정규화 완성도는 미확인. 오답 생성이 "같은 품사+같은 급수"까지 엄밀히 되는지는 데이터 정제 상태에 의존.
- **일일 상한/어뷰징(Play Integrity), 광고 보너스 상자, 잠금화면 학습, 예문/듣기, 푸시, iOS** = 계획서 2·3차 그대로 미착수.
- `QuizLog` 7일 자동삭제 배치(`learning/tasks.py`)의 실제 스케줄링(celery beat 등) 연결 여부 미확인.
