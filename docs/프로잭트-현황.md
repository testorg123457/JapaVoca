## 1. 한 줄 요약

일본어(단어·한자·가나) 학습 + 캐시 리워드 안드로이드 앱. **퀴즈 정답 → 캐시상자 획득 → (광고 보고) 개봉 → 캐시 적립 → 기프티콘 교환(광고)** 이 핵심 루프. 수익 모델은 캐시워크형(AdMob 보상형 광고). 캐시는 **자산**이라 정합성(원장+잔액 트랜잭션)이 최우선 원칙.

- 서비스명: 일본어 한자 보카 / JapaVoca / 패키지 `com.japavoca.app`
- 1차 플랫폼: **Android 우선** (iOS 3차)

## 2. 모노레포 구조 & 스택

```
JapaVoca/
├── apps/
│   ├── mobile/   # React Native 0.86 (TS, New Architecture). android/ 에 네이티브(Kotlin)
│   └── server/   # Django 6 + DRF (자체 .venv), PostgreSQL
└── docs/
```
- JS workspace 도구(Nx/Turbo/yarn workspaces) **안 씀**. mobile/server는 같은 레포의 독립 폴더.
- 백엔드는 Python venv로 독립.

| 레이어 | 채택 |
|---|---|
| 클라이언트 | React Native 0.86 (TS), New Architecture(Fabric/Bridgeless), Hermes |
| UI | **NativeWind(Tailwind) + 자체 토큰** (⚠️ gluestack-ui는 제거됨) |
| 상태/데이터 | React Query(서버 상태) + React Context(auth) + MMKV(토큰 영속) |
| 애니메이션 | reanimated(+worklets), gesture-handler, lottie |
| 광고 | react-native-google-mobile-ads (AdMob 보상형) |
| 인증 | 구글 소셜 로그인(@react-native-google-signin) + **게스트 로그인** + JWT |
| 백엔드 | Django + DRF, SimpleJWT, 커스텀 User(google_uid) |
| DB | PostgreSQL. 테이블명은 전부 `tbl_<app>_<model>` 규칙 |

## 3. 백엔드 아키텍처 (apps/server)

루트 라우팅(`config/urls.py`): `/api/auth/` `/api/quiz/` `/api/rewards/` `/api/exchange/` `/api/notifications/`

### 앱별 책임
| 앱 | 책임 | 주요 모델 |
|---|---|---|
| `accounts` | 구글·게스트 로그인·JWT 발급, 프로필, 약관동의 | `User`, `ConsentAgreement` |
| `content` | 학습 콘텐츠 (URL 없음 — 퀴즈 통해서만 노출) | `Kanji`, `Word`, `WordMeaning`, `Kana` |
| `learning` | 퀴즈 출제(SRS+오답생성)·채점(SM-2) | `SrsState`, `QuizLog` |
| `rewards` | 캐시(지갑/원장)·상자·출석·데일리 | `Wallet`, `Ledger`, `CashBox`, `Attendance`, `Daily` |
| `exchange` | 기프티콘 교환·광고 SSV 검증 | `GiftExchange`, `AdRewardLog` |
| `notifications` | 인앱 알림 | `Notification` |

### 캐시 정합성 핵심 (⚠️ 절대 원칙 — CLAUDE.md)
- 모든 캐시 변동 = **Wallet 잔액 갱신 + Ledger(append-only) 기록을 하나의 `@transaction.atomic`**.
- `wallet.balance == (ledger earn합 − use합)` 항상 일치.
- 차감은 행 잠금(`_lock_wallet`) + 조건부로 음수/이중차감 방지. `Ledger`는 수정·삭제 금지.
- 금액은 전부 정수 캐시(BigInteger), 소수점 없음.
- `rewards.services`: `earn()`, `use()`, `open_cash_box()`, `check_in()`, `get_today_*()`.

### 엔드포인트 맵 (현재)
| 메서드·경로 | 설명 |
|---|---|
| `POST /api/auth/guest/` | 기기 UUID → 게스트 JWT |
| `POST /api/auth/google/` | 구글 ID 토큰 → JWT |
| `POST /api/auth/link/` | 게스트 → 소셜 계정 연결 |
| `POST /api/auth/token/refresh/` | 토큰 갱신 |
| `GET/PATCH /api/auth/me/` | 프로필 조회/수정 (study_mode·study_level·study_kana_hiragana·study_kana_katakana 포함) |
| `GET /api/auth/consent/status/` | 약관 동의 현황 |
| `POST /api/auth/consent/` | 약관 동의 제출 |
| `GET /api/quiz/next/` | 다음 문제 1개(서명된 question_token 포함) |
| `POST /api/quiz/answer/` | 서버 채점 + SRS 갱신 + QuizLog + Daily + 정답 시 상자 생성 |
| `GET /api/rewards/wallet/` · `ledger/` · `boxes/` | 잔액 / 내역 / 미개봉 상자 |
| `POST /api/rewards/boxes/<id>/open/` | 상자 개봉(보상 확정 + earn 원자적) |
| `GET /api/rewards/attendance/today/` · `daily/today/` | 출석 현황·체크인 / 오늘 학습현황 |
| `GET /api/exchange/products/` · `POST request/` · `GET history/` | 상품목록 / 교환요청 / 교환내역 |
| `GET /api/exchange/admob/ssv/` | AdMob SSV 콜백 검증 |
| `GET /api/notifications/` | 인앱 알림 목록 |

### SRS / 채점 (`learning.services`)
- 출제: SRS 기한 도래 항목 우선 → 없으면 미학습 신규(가능하면 유저 급수). 오답은 같은 종류 다른 항목에서 추출.
- 문제마다 **서명된 question_token** 발급 → 채점 시 위조/만료 검증.
- 채점: 간이 **SM-2**(ease/interval/repetitions/due_at) 갱신, 정답 시 상자 생성(일일 상한 체크).
- **학습 트랙**: `kanji`(한자) / `kanji_word`(한자단어) / `kana_word`(가나단어) / `kana`(가나글자).
- 가나 트랙: `Kana` 모델(~208행, hira/kata × 청음·탁음·요음), 글자→로마자 방향 고정(로마자→글자는 じ·ぢ 동음 모호성 때문).
- `QuizLog`는 **7일 보관**(배치 삭제 예정), 장기 통계는 `rewards.Daily` 집계로 보존.

## 4. 프론트 아키텍처 (apps/mobile/src)

### 내비게이션 (`navigation/`)
```
RootNavigator
├── Onboarding (게이트 조건 미충족 시)
│   ├── Terms         ← 약관 동의
│   ├── TermsDetail
│   ├── Permissions   ← 알림·전화·배터리 최적화 권한
│   └── StudySelect   ← 학습 트랙 선택
└── Main (게이트 통과 시)
    ├── Home      ← 단일 스크롤(구 BottomTab 대체)
    ├── Quiz
    ├── BoxOpen   (modal)
    ├── Kana      ← 가나표 보기
    ├── Attendance
    ├── Settings
    ├── Ledger / ExchangeHistory / Exchange / Notifications
    └── StyleGuide (DEBUG)
```

**온보딩 게이트** (`navigation/gateStatus.ts`): `computeGateStatus()` 순수 함수로 단계 결정.
`consentLoading → terms → permissions → studyNeeded → ready`

### API 레이어 (`api/`)
- `client.ts`: 공용 axios. access 토큰 자동 주입, **401 → refresh 1회 재시도**, refresh 실패 시 `forceSignOut()`.
- `hooks.ts`: React Query 훅 모음. `useMe`, `useWallet`, `useLedger` 등.
- `quiz.ts` / `boxes.ts` / `exchange.ts`: 퀴즈·상자·교환 관련 훅.

### 디자인 (`theme/tokens.ts` 단일 소스 + NativeWind)
- 색·간격·라운드·그림자·타이포를 토큰에서만 정의(하드코딩 금지). 톤은 **둥글둥글**, 포인트 컬러 절제, 폰트 Pretendard.
- 공용 컴포넌트 `components/`: `AppText`/`Button`/`Card`/`CashBadge` + `Icon`(SVG)/`Tag`/`SectionHeader`/`ProgressBar`/`Gradient`/`PressableScale`/`StudySelector`.
- `StyleGuideScreen`으로 전체 컴포넌트 확인 가능(설정 화면 DEBUG 진입점).

## 6. 핵심 루프 현 구현 정리

온보딩(✅) → 로그인(✅) → 홈(✅) → 퀴즈 정답(✅) → 상자 생성(서버 ✅) → 상자 개봉+광고(🟡) → 캐시 적립(서버 earn ✅) → 지갑(🟡) → 기프티콘 교환+광고(서버 로직 ✅ / 발급사 Mock) → SSV 검증(✅).

**서버 비즈니스 로직은 대부분 구현**. 프론트는 로그인·온보딩·홈·퀴즈까지 검증됨. 상자 개봉 이후 화면은 코드만 있고 실제 플로우 검증 전.

## 7. 주요 변경 이력 & 미구현

**초안 대비 바뀐 결정**
- **gluestack-ui 제거**: 웹 react-dom 의존으로 RN 번들 깨짐 → NativeWind + 자체 컴포넌트로 대체.
- **게스트 로그인 추가**: 구글 OAuth 없이 UUID로 즉시 진입. 추후 소셜 계정 연결 가능(`/api/auth/link/`).
- **BottomTab 제거**: 하단 3탭(홈/지갑/설정) → 홈 단일 스크롤로 통합. 지갑·설정은 홈에서 진입.
- **온보딩 게이트 시스템**: 약관·권한·학습선택을 순서대로 강제하는 `computeGateStatus()` 함수 기반 구조.
- **가나 학습 트랙 추가**: 4번째 트랙(`kana`). `Kana` 모델(DB) + SRS + 글자→로마자 퀴즈 + 보상.
- **배터리 최적화 권한**: 잠금화면 학습을 위해 필수 권한으로 추가. 네이티브 모듈(`BatteryOptimizationModule.kt`).

**아직 미구현 (MVP 갭)**
- **기프티콘 발급사 = Mock** (`exchange/providers.py`). 실제 B2B API 미연동.
- **문의하기**: 현재 Gmail Alert → DB 방식으로 교체 예정. 계획서 → `docs/문의하기-구현플랜.md`.
- **Play Integrity·어뷰징 방어**: 일일 상한(`MAX_BOXES_PER_DAY=50`)만 있고 응답속도 탐지·기기 검증은 미착수.
- **광고 보너스 상자, 잠금화면 학습, 예문/듣기, 푸시, iOS** = 2·3차 미착수.
- **가나 시드 데이터**: `content/sql/seed_kana.sql` 생성됨, 아직 DB에 적용 전. 스키마 마이그레이션(`content.0004_kana`)도 `python manage.py migrate` 실행 전.
- `QuizLog` 7일 자동삭제 배치: Celery 제거 후 Cloud Scheduler + Cloud Run Job 방식으로 연결 예정. 현재 미연결.
- **앱 release 빌드 미완료**: 현재 debug 빌드만 있어 Metro 번들러 필요. 독립 실행하려면 release APK 빌드 필요.

## 8. 실행 방법

[`로컬-실행-가이드.md`](로컬-실행-가이드.md) 참고. 요약:
- **백엔드**: `source .venv/bin/activate && python manage.py runserver 0.0.0.0:8001` (포트 8000은 Docker가 점유, 8001 사용)
- **에뮬레이터**: `API_BASE_URL=http://10.0.2.2:8001` (`.env`)
- **실기기(USB)**: `adb reverse tcp:8001 tcp:8001` 또는 `.env`에 Mac WiFi IP 직접 입력
- **첫 실행 시 `python manage.py migrate` 필수** (pending 마이그레이션: `content.0004_kana`, `learning.0004_*`)
- 구글 로그인 키 없으면 **"개발용 로그인 (DEBUG)"** 또는 **게스트 로그인** 으로 진입

## 10. 디자인 시스템 ⚠️ (UI 작업 전 필독)

> 실물은 `StyleGuideScreen`(설정 → 디자인 시스템, DEBUG 전용). 규칙 상세는 [`디자인-시스템-원칙.md`](디자인-시스템-원칙.md).

### 핵심 원칙
1. **이모지를 기능 아이콘으로 쓰지 않는다.** 내비/상태/구조에는 `Icon`(SVG, `components/Icon.tsx`)만. 이모지는 "보상 축하" 같은 연출에만 제한적으로.
2. **색은 기능적으로만.** 민트=주 액션/활성, 옐로=캐시/리워드, 회색=중립, danger=경고. 장식 목적 색 금지.
3. **위계는 크기 + 굵기로.** Bold(hero/title) → SemiBold(heading/label) → Regular(body/caption).
4. **여백이 숨쉬게.** 섹션 간 `gap-2xl`(20)+, 카드 패딩 `p-xl`(16).
5. **색 인라인은 `useThemeColors()`, className은 semantic 토큰**(`bg-bg-primary`/`text-text-secondary`/`bg-brand`…).

### 토큰 (`theme/tokens.ts`)
- **폰트**: Pretendard 4종. ⚠️ 폰트는 네이티브 에셋이라 **`run-android` 재빌드해야 적용**.
- **typography variant**: `hero 42 / display 28 / title 22 / heading 18 / subheading 16 / body 15 / label 14 / caption 13 / micro 11`.
- **radius**: sm 10 / md 14(버튼) / lg 18(카드) / xl 24(시트·큰카드) / 2xl 28 / full.
- **gradients.brand** `['#2FC9C3','#1AA6A0']` — hero(캐시 잔액) 카드용.

### 공용 컴포넌트 (`components/`)
| 컴포넌트 | 용도 |
|---|---|
| `AppText` | variant(타이포). 기본색 text-primary |
| `Button` | variant(filled/soft/outline/ghost) · size · `loading` · `leftIcon` |
| `Card` | variant(default/elevated/flat/brand) · `onPress` |
| `Icon` | SVG 아이콘 (20여 종) |
| `Tag` | variant(brand/amber/neutral/success/danger) — 칩/상태 |
| `CashBadge` | SVG 코인 + 금액 pill |
| `StudySelector` | 학습 트랙 선택 UI (온보딩/설정 공유) |
| `PressableScale` | 누르면 scale 0.97~0.98 Pressable 기반 |
