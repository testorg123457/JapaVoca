# 퀴즈 로직

> 구현 위치: `apps/server/learning/services.py`

---

## 전체 흐름

```
GET /api/quiz/next/
  → resolve_study()       유저 study_mode → 트랙 결정
  → _pick_item_id()       출제 항목 선택 (SRS 우선 → 신규)
  → build_question()      문제 방향·보기 생성 → 서명 토큰 발급
  → 클라이언트에 문제 전달

POST /api/quiz/answer/
  → grade_answer()        토큰 검증 → 채점 → SRS 갱신 → 상자 생성
```

---

## 1. 트랙 결정 (`resolve_study`)

유저의 `study_mode` 값에 따라 출제 대상을 결정한다.

| study_mode | 출제 대상 | 급수 필터 |
|---|---|---|
| `kanji` | `Kanji` 모델 (한자) | `study_level` |
| `kanji_word` | `Word` (word_type=kanji) | `study_level` |
| `kana_word` | `Word` (word_type=kana) | `study_level` |
| `kana` | `Kana` 모델 (가나 글자) | 없음 (히라/가타 토글) |

`kana` 트랙은 `study_kana_hiragana` / `study_kana_katakana` 플래그로 히라가나·가타카나 중 어느 쪽을 출제할지 결정한다. 둘 다 false면 `NoContent` 에러.

---

## 2. 항목 선택 (`_pick_item_id`)

해당 트랙 항목 중 아래 우선순위로 1개 선택한다.

1. **SRS 복습 기한 도래 항목** — `due_at ≤ now` 중 가장 오래된 것
2. **신규 항목** — 한 번도 본 적 없는 것 중 유저 급수(`jlpt_level`)에 맞는 것 우선, 없으면 급수 무관 랜덤

> `kanji_word` ↔ `kana_word` 트랙 간 오염 방지: SRS 조회 시에도 현재 트랙의 `word_type`으로 제한한다.

---

## 3. 문제 생성 (`build_question`)

### 문제 방향

| 트랙 | 방향 |
|---|---|
| 가나 | 항상 **글자 → 로마자** (반대 방향은 じ·ぢ 같은 동음이자로 정답 모호) |
| 한자 / 단어 | 랜덤으로 둘 중 하나 |

- `word_to_meaning`: 표기(한자·단어·가나) 제시 → 뜻(또는 로마자) 맞추기
- `meaning_to_word`: 뜻 제시 → 표기 맞추기

`meaning_to_word`일 때는 `reading`(읽기)을 내려보내지 않는다. 읽기가 곧 정답 힌트가 되기 때문.

### 오답 보기 생성 (`_distractor_texts`)

같은 종류(한자면 한자끼리, 단어면 단어끼리)에서 랜덤 60개를 뽑아 정답과 다른 텍스트 3개를 추출한다. 정답 포함 4개를 섞어 4지선다를 완성한다.

### 서명 토큰 발급

정답 인덱스(`correct_index`)를 Django signing으로 암호화해 `question_token`으로 발급한다.

```python
payload = {'it': item_type, 'id': item_id, 'qt': question_type, 'ci': correct_index}
token = signing.dumps(payload, salt='japavoca.quiz.question')
```

- TTL: **5분**
- 클라이언트는 정답 인덱스를 알 수 없다 (보안 원칙: 정답 판정은 서버에서만).

## 4. 채점 (`grade_answer`)

```python
@transaction.atomic
def grade_answer(user, question_token, choice_index, answer_ms=None):
```

### 검증

1. 토큰 서명 검증 + 만료(5분) 확인 → 실패 시 `InvalidQuestionToken`
2. `token_hash`(SHA-256) unique 제약으로 **중복 채점 차단** (더블탭·리플레이 방지)

### SRS 갱신 (SM-2)

| 상태 | 정답 | 오답 |
|---|---|---|
| `repetitions` | +1 | 0으로 리셋 |
| `interval_days` | 1일 → 6일 → × ease | 1일로 리셋 |
| `ease` | 최대 2.5까지 증가 | 최소 1.3까지 감소 |
| `due_at` | `now + interval_days` | `now + 1일` |

### 캐시상자 생성

정답이고 오늘 획득 상자 수가 **일일 상한(50개) 미만**이면 상자를 1개 생성한다.

**상자 등급 확률**

| 등급 | 확률 |
|---|---|
| NORMAL | 70.0% |
| RARE | 14.0% |
| EPIC | 11.5% |
| LEGENDARY | 4.0% |
| JACKPOT | 0.5% |

### Daily 집계

`quiz_count`, `correct_count`, `boxes_earned`를 원자적으로 갱신한다 (행 잠금 `select_for_update`).

---

## 모델 관계

```
User
 ├─ SrsState (item_type, item_id, due_at, ease, interval_days, repetitions)
 ├─ QuizLog  (token_hash unique, is_correct, answer_ms, box FK)
 ├─ Daily    (date, quiz_count, correct_count, boxes_earned)
 └─ CashBox  (grade, opened_at)
```

`QuizLog`는 7일 보관 후 자동 삭제 예정 (`learning.management.commands.delete_old_quiz_logs`). 장기 통계는 `Daily`로 보존.
