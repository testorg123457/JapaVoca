## 전체 흐름

### 세트 모드 (메인 앱 — 10문제 세트)

```
GET /api/quiz/set/
  → build_quiz_set()
      ① 활성 세트(completed_at IS NULL, abandoned_at IS NULL) 있으면 반환
      ② 쿨다운 체크 (마지막 완료 후 30초) → cooldown_until 반환
      ③ resolve_study() → _pick_item_id() × 10 → 세트 생성 → 서명 토큰 발급

POST /api/quiz/answer/   (세트 토큰)
  → grade_answer()        토큰 검증 → 채점 → SRS 갱신 → 상자 생성
                          10문제 모두 완료 시 completed_at 기록

POST /api/quiz/set/abandon/
  → abandon_quiz_set()    활성 세트의 abandoned_at = now() (쿨다운 미발동)
```

### 단발 모드 (잠금화면 등)

```
GET /api/quiz/next/
  → resolve_study()       유저 study_mode → 트랙 결정
  → _pick_item_id()       출제 항목 선택 (SRS 우선 → 신규)
  → build_question()      문제 방향·보기 생성 → 서명 토큰 발급 (TTL 5분)
  → 클라이언트에 문제 전달

POST /api/quiz/answer/
  → grade_answer()        토큰 검증 → 채점 → SRS 갱신 → 상자 생성
```

## 1. 트랙 결정 (`resolve_study`)

유저의 `study_mode` 값에 따라 출제 대상을 결정한다.

| study_mode | 출제 대상 | 급수 필터 |
|---|---|---|
| `kanji` | `Kanji` 모델 (한자) | `study_level` |
| `kanji_word` | `Word` (word_type=kanji) | `study_level` |
| `kana_word` | `Word` (word_type=kana) | `study_level` |
| `kana` | `Kana` 모델 (가나 글자) | 없음 (히라/가타 토글) |

`kana` 트랙은 `study_kana_hiragana` / `study_kana_katakana` 플래그로 히라가나·가타카나 중 어느 쪽을 출제할지 결정한다. 둘 다 false면 `NoContent` 에러.

---

## 2. 항목 선택 (`_pick_item_id`)

해당 트랙 항목 중 아래 우선순위로 1개 선택한다.

1. **SRS 복습 기한 도래 항목** — `due_at ≤ now` 중 가장 오래된 것
2. **신규 항목** — 한 번도 본 적 없는 것 중 유저 급수(`jlpt_level`)에 맞는 것 우선, 없으면 급수 무관 랜덤

> `kanji_word` ↔ `kana_word` 트랙 간 오염 방지: SRS 조회 시에도 현재 트랙의 `word_type`으로 제한한다.

> `kanji` 트랙에서 `jlpt_level`이 없는 한자(NULL 또는 공백)는 출제에서 제외한다. 급수 정보가 없는 항목은 난이도 필터를 적용할 수 없어 학습 흐름에 맞지 않기 때문.

## 3. 문제 생성 (`build_question`)

### 문제 방향

| 트랙 | 방향 |
|---|---|
| 가나 | 항상 **글자 → 로마자** (반대 방향은 じ·ぢ 같은 동음이자로 정답 모호) |
| 한자 / 단어 | 랜덤으로 둘 중 하나 |

- `word_to_meaning`: 표기(한자·단어·가나) 제시 → 뜻(또는 로마자) 맞추기
- `meaning_to_word`: 뜻 제시 → 표기 맞추기

`meaning_to_word`일 때는 `reading`(읽기)을 내려보내지 않는다. 읽기가 곧 정답 힌트가 되기 때문.

### 오답 보기 생성 (`_distractor_texts`)

같은 종류(한자면 한자끼리, 단어면 단어끼리)에서 랜덤 60개를 뽑아 정답과 다른 텍스트 3개를 추출한다. 정답 포함 4개를 섞어 4지선다를 완성한다.

### 서명 토큰 발급

정답 인덱스(`correct_index`)를 Django signing으로 암호화해 `question_token`으로 발급한다.

```python
# 단발 토큰 (TTL 5분)
payload = {'it': item_type, 'id': item_id, 'qt': question_type, 'ci': correct_index}

# 세트 토큰 (TTL 없음 — set_id 생존으로 검증)
payload = {'it': item_type, 'id': item_id, 'qt': question_type, 'ci': correct_index,
           'sid': quiz_set.id, 'ord': order}

token = signing.dumps(payload, salt='japavoca.quiz.question')
```

- 클라이언트는 정답 인덱스를 알 수 없다 (보안 원칙: 정답 판정은 서버에서만).

## 4. 채점 (`grade_answer`)

```python
@transaction.atomic
def grade_answer(user, question_token, choice_index, answer_ms=None):
```

### 검증

1. 토큰 서명 검증 → 실패 시 `InvalidQuestionToken`
   - **단발 토큰**: 추가로 만료(5분) 확인
   - **세트 토큰** (`sid` 포함): TTL 없음 — QuizSet 존재 여부로 대체 검증
2. `token_hash`(SHA-256) unique 제약으로 **중복 채점 차단** (더블탭·리플레이 방지)
   - 세트 토큰은 `QuizSetItem.answered` 플래그로 중복 방지 (token_hash 체크 생략)

### SRS 갱신 (SM-2)

| 상태 | 정답 | 오답 |
|---|---|---|
| `repetitions` | +1 | 0으로 리셋 |
| `interval_days` | 1일 → 6일 → × ease | 1일로 리셋 |
| `ease` | 최대 2.5까지 증가 | 최소 1.3까지 감소 |
| `due_at` | `now + interval_days` | `now + 1일` |

### 캐시상자 생성

정답이고 오늘 획득 상자 수가 **일일 상한(50개) 미만**이면 상자를 1개 생성한다.

**상자 등급 확률**

| 등급 | 확률 |
|---|---|
| NORMAL | 70.0% |
| RARE | 14.0% |
| EPIC | 11.5% |
| LEGENDARY | 4.0% |
| JACKPOT | 0.5% |

### Daily 집계

`quiz_count`, `correct_count`, `boxes_earned`를 원자적으로 갱신한다 (행 잠금 `select_for_update`).

## 5. 세트 관리 (`build_quiz_set` / `abandon_quiz_set`)

### 세트 상태

| 상태 | 조건 |
|---|---|
| 활성 | `completed_at IS NULL AND abandoned_at IS NULL` |
| 폐기 | `abandoned_at IS NOT NULL` (completed_at은 NULL 유지) |
| 완료 | `completed_at IS NOT NULL` |

### 쿨다운

완료(`completed_at`) 기준으로만 30초 쿨다운이 적용된다. **폐기(`abandoned_at`)는 쿨다운 대상이 아니다.**

→ 설정 화면에서 모드/난이도를 바꾸고 "새 설정으로 시작"을 누르면 현재 세트가 폐기되고 즉시 새 세트를 받을 수 있다.

### 세트 내 상자 상한

세트당 최대 **3개** 상자, 하루 전체 **50개** 상한. `SET_BOX_CAP = 3`, `MAX_BOXES_PER_DAY = 50`.

## 부록 — SRS(간격 반복 학습)란?

### 개념

SRS는 **망각 곡선을 역이용하는 학습법**이다. 사람의 기억은 시간이 지날수록 지수적으로 흐려지는데, 이 "막 잊으려는 순간"에 딱 복습을 넣으면 최소한의 반복으로 최대한 오래 기억에 남길 수 있다.

```
기억 강도
100% ┤
     │╲  복습 →  ╲  복습 →    ╲  복습 →
     │  ╲         ╲             ╲
  0% ┤   ───────── ─────────────  ────────
                  시간 →
```

자주 틀리는 항목은 짧은 간격으로 자주 등장하고, 잘 아는 항목은 간격을 점점 늘려 장기 기억으로 굳힌다.

---

### JapaVoca에서의 구현 — SM-2 알고리즘

`SrsState` 테이블(`tbl_learning_srsstate`)에 유저 × 학습항목 단위로 상태를 저장한다.

**저장 필드:**

| 필드 | 설명 |
|---|---|
| `due_at` | 다음 복습 예정 시각 — 이 시간이 지나야 출제 대상이 됨 |
| `interval_days` | 현재 복습 간격(일) |
| `ease` | 난이도 계수 (기본 2.5, 범위 1.3~∞) |
| `repetitions` | 연속 정답 횟수 |
| `last_result` | 마지막 결과 (correct / wrong) |

**채점 후 갱신 로직 (`_apply_sm2`):**

```
정답일 때
  repetitions = 0 → interval = 1일
  repetitions = 1 → interval = 6일
  repetitions ≥ 2 → interval = interval × ease (반올림)
  repetitions += 1
  ease = ease + 0.1 - (5-q) × (0.08 + (5-q) × 0.02)   (q=5)
         → 정답 시 ease가 소폭 상승

오답일 때
  interval = 1일로 초기화
  repetitions = 0으로 초기화
  ease = max(1.3, ease - 0.2)   → 오답마다 ease 감소

due_at = now + interval_days
```

**실제 진행 예시 (ease=2.5 유지 가정):**

| 회차 | 결과 | interval | 다음 복습 |
|---|---|---|---|
| 1회 | 정답 | 1일 | 내일 |
| 2회 | 정답 | 6일 | 6일 후 |
| 3회 | 정답 | 15일 (6 × 2.5) | 15일 후 |
| 4회 | 정답 | 38일 (15 × 2.5) | 38일 후 |
| - | 오답 | 1일로 리셋 | 내일 |

### 출제 우선순위와의 연결

`_pick_item_id`에서 SRS가 실제로 작동하는 방식:

```
1순위: due_at ≤ 지금인 항목 → 복습 기한 도래 (가장 오래된 것부터)
2순위: SrsState 레코드 자체가 없는 항목 → 처음 보는 신규 문제
```

즉 한 번도 안 본 단어보다 **"오늘 복습해야 할 단어"가 먼저** 나온다. 신규 항목은 복습 기한이 쌓이지 않은 경우에만 채워진다.
