# 퀴즈 개편 — 화면 구조 (UX 명세)

> 퀴즈 페이지를 "단일 문제 무한 루프" → **"10문제 세트 + 풀이 후 정답 해설 패널"** 구조로 개편한다.
> 이 문서는 **무엇이 어디에 있고, 누르면 무엇이 뜨는가**(구조/UX)만 다룬다.
> 구현 방법은 [`퀴즈-개편-구현플랜.md`](퀴즈-개편-구현플랜.md), 작업 규칙은 [`CLAUDE.md`](../CLAUDE.md), 현황은 [`프로잭트-현황.md`](프로잭트-현황.md), 출제/채점 로직은 [`퀴즈-로직.md`](퀴즈-로직.md).

---

## 1. 현재 구조 (개편 전)

- 메인 퀴즈 화면 = `apps/mobile/src/screens/quiz/LockQuizScreen.tsx`의 `LockQuizView` (잠금화면 다크 톤).
- 흐름: `getNextQuiz()`로 **문제 1개** 받음 → 선택 → `submitAnswer()` 채점 → **1.2초 정/오답 색만 표시**(`FEEDBACK_MS=1200`) → 자동으로 다음 문제 로드. 무한 반복.
- 화면 구성: 상단 시계·상자 배지 / 중앙 (유형 라벨 + 읽기 + 문제 + 지시문) / 2×2 선택지 / 하단 "밀어서 잠금해제".
- 상자: 정답마다 서버가 생성, **일일 50개 상한**(`MAX_BOXES_PER_DAY`). 온라인 필수.

### 개편하는 이유 (브레인덤프 4가지)
1. **오프라인에서도** 문제를 풀 수 있어야 함. 단, 오프라인이면 **상자는 안 줌**.
2. 문제는 **약 10개 세트**로 생성. 다 풀면 **1시간 주기로 재생성**. 상자 획득은 **세트(1시간)당 최대 3개**.
3. 퀴즈 화면에 **음성 듣기 버튼**(한자·히라가나·가타카나 발음).
4. 정답 선택 시 자동으로 넘어가지 말고 **해설 패널**을 띄움 — "정답입니다!"(초록) + 부수/구성자 + 음성 + **북마크 버튼**.

## 2. 출제 모델 — 10문제 세트 / 1시간 재생성 / 상자 3개

```
세트 발급(10문제)  →  1~10번 순서대로 풀이  →  10번 완료
                                                  │
                                          (완료 시각 + 1시간)
                                                  ▼
                                          다음 세트 발급 가능
```

- **세트 = 10문제 묶음.** 발급 시 10문제 전체(문제·선택지·해설데이터·정답키)를 한 번에 내려받아 **로컬 캐시**(오프라인 대비).
- **진행은 순차.** 한 문제 풀면 해설 패널 → "다음" → 다음 문제. 세트 안에서는 앞뒤 이동 없이 1→10 진행.
- **세트 완료(10번 풀이) 시점부터 1시간 쿨다운.** 쿨다운 중에는 새 세트 없음 → 홈/퀴즈 진입 시 "다음 세트까지 남은 시간" 표시.
- **세트를 다 안 풀면** 그 세트가 계속 "현재 세트"로 유지(쿨다운 시작 안 함). 이어풀기 가능.
- **상자 캡: 세트당 최대 3개.** 세트 내 정답 중 (온라인 상태로 푼) 처음 3개에만 상자 지급. 4번째 정답부터는 상자 없음(해설은 정상 표시). 기존 일일 50개 상한은 상위 안전장치로 유지.

### 상자 카운터 표시
- **우상단 상자 배지(현행 유지)** = 보유 미개봉 상자 개수. 손대지 않음.
- 세트 3개 캡 자체는 별도 상단 카운터로 노출하지 않는다(크롬 최소 변경). 대신:
  - 정답 시 해설 패널의 **"상자 +1" 칩**으로 지급을 알림.
  - 3개를 다 채운 뒤 정답엔 칩 대신 "이번 세트 상자를 모두 받았어요"(약하게).

## 3. 퀴즈 풀이 화면 (문제 단계)

**기존 잠금화면 크롬(`LockQuizView`)을 그대로 유지하고, 중앙 퀴즈 영역만 교체한다.**
즉 좌상단 시계/날짜, 우상단 앱 열기·상자 배지, 하단 "밀어서 잠금해제"는 **현행 그대로**.

```
┌────────────────────────────────────────────┐
│  09:41                       앱 열기 ›  [🎁 3] │  ← (유지) 시계 / 앱열기 · 상자수
│  9월 12일 목요일                               │  ← (유지) 날짜·요일
│                                              │
│ ┄┄┄┄┄┄┄┄┄ 중앙: 퀴즈 영역(교체) ┄┄┄┄┄┄┄┄ │
│                                              │
│                    3 / 10                    │  ← 진행 (옅게)
│                     ねん                       │  ← 읽기 (word_to_meaning일 때만)
│                                              │
│                      年                       │  ← 문제(prompt, 크게)
│                   ( 🔊 듣기 )                  │  ← 음성 버튼 (prompt 아래)
│                                              │
│            다음 한자의 뜻으로 옳은 것은?         │  ← 지시문
│                                              │
│      ┌──────────┐   ┌──────────┐             │
│      │   해     │   │   달     │             │  ← 2×2 선택지
│      └──────────┘   └──────────┘             │
│      ┌──────────┐   ┌──────────┐             │
│      │   년     │   │   주     │             │
│      └──────────┘   └──────────┘             │
│ ┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄ │
│                                              │
│              밀어서 잠금해제 ›                  │  ← (유지) 하단 스와이프
│              ▬▬▬▬▬▬▬▬                        │
└────────────────────────────────────────────┘
```

### 유지(현행 그대로) — 손대지 않음
- **좌상단**: 시계(시:분, hero) + 그 아래 날짜·요일.
- **우상단**: "앱 열기" 고스트 버튼 + 상자 배지(보유 미개봉 상자 개수).
- **하단**: "밀어서 잠금해제"(화면 어디서나 오른쪽 스와이프) + 핸들 바.
- 다크 잠금화면 팔레트(`LOCK`)·배경 그라데이션·제스처.

### 변경 — 중앙 퀴즈 영역만

위→아래 순서:

1. **진행 표시 `3 / 10`** — 옅은 텍스트. (기존 **유형/급수 칩은 제거**.)
2. **읽기(reading)** — `word_to_meaning`일 때만. (현행 유지)
3. **문제(prompt)** — 크게(hero). (현행 유지)
4. **음성(🔊) 버튼** — prompt 바로 아래. (신규)
5. **지시문** — "다음 한자의 뜻으로 옳은 것은?" 등. (현행 유지)
6. **2×2 선택지** — 번호 없이 텍스트 중앙. (현행 유지)

> ❌ 제거: 좌상단 **유형/급수 칩**(`한자 · N3`). 더 이상 표시하지 않는다.

### 음성(🔊) 버튼 (신규)
- 위치: **문제(prompt) 바로 아래**. 작은 pill 버튼.
- 누르면: 문제 대상의 발음 재생.
  - 한자 → 음독/훈독 대표 읽기 발음.
  - 가나(히라가나·가타카나) → 글자 발음.
  - 단어 → 단어 읽기(reading) 발음.
- **출제 방향에 따른 노출 규칙**(정답 누설 방지):
  - `word_to_meaning`(표기→뜻): **노출**(읽기는 힌트지 정답 아님).
  - `meaning_to_word`(뜻→표기): 음성이 정답 표기 발음이라 **풀기 전 숨김** → 해설 패널에서만 노출.
  - 가나: 글자→로마자 고정이라 풀이 단계 노출 OK.
- 재생 불가(기기 일본어 음성 없음/오프라인 미지원): **비활성(흐리게)** + 탭 시 "이 기기에서 음성을 지원하지 않아요" 토스트.

### 선택지
- 2×2 그리드, 번호 없이 텍스트 중앙. (현행 유지)
- 선택 즉시 정/오답 색 + 체크/엑스 아이콘. **자동으로 넘어가지 않고** 4번 해설 패널로 전환.

## 4. 해설 패널 (정답/오답 후) — 핵심 신규

선택지를 고르면 **하단에서 패널이 올라오거나** 화면 하단 영역이 해설로 채워진다(자동 넘김 제거).

```
┌────────────────────────────────────────────┐
│   (문제 + 정답 표시된 선택지는 위에 그대로)      │
├────────────────────────────────────────────┤
│  ✓ 정답입니다!            [☆ 북마크] [🔊]    │  ← 결과 헤더 + 액션
│                                              │
│   年  ねん / とし   N3                        │  ← 표기 · 읽기 · 급수
│   뜻: 해, 년                                  │
│   ───────────────────────────                │
│   부수/구성자                                 │
│   ⼲(방패 간) + ㇒ …                          │  ← 한자/한자단어만
│   획수 6획                                    │
│                                              │
│              [   다음 문제   ]                │  ← 진행 버튼
└────────────────────────────────────────────┘
```

### 4-1. 결과 헤더
- 정답: **초록** "✓ 정답입니다!".
- 오답: **빨강/주황** "✗ 아쉬워요" + 정답이 무엇이었는지 강조(선택지에 이미 색 표시).
- 상자 획득 시: 헤더 옆/아래에 작은 **"상자 +1"** 칩(잠깐 반짝).
- 칩 미표시 조건 — 아래 사유 중 하나에 해당하면 칩 없음 + 하단에 작은 안내 문구:
  - 오프라인: "오프라인이라 상자가 지급되지 않았어요"
  - 세트 3개 캡 소진: "이번 세트 상자를 모두 받았어요"
  - 일일 50개 상한 초과: "오늘 받을 수 있는 상자를 모두 받았어요"
  - (오프라인 + 캡 소진이 겹치면 오프라인 사유만 표시.)

### 4-2. 해설 본문 (항목 타입별로 내용 다름 — 6절 참고)
- 공통: **표기 · 읽기 · 뜻 · 급수**.
- **부수/구성자**: 한자 / 한자단어일 때만. Kanji `components` 데이터 사용(예: `口(입 구) + 𠂇(왼 좌)`). **획수**도 함께.
- 가나: 부수/구성자 없음 → **글자 · 로마자 · (히라/가타 구분)**만.
- 단어(가나단어): 구성자 없음 → 표기·읽기·뜻.

### 4-3. 액션 버튼
- **북마크 (☆/★)**: 토글. 이 항목(한자/단어/가나)을 내 북마크에 저장/해제. **오프라인에서도 토글 가능**(로컬 저장 후 온라인 시 동기화). 별 아이콘 채움으로 상태 표시.
- **음성 (🔊)**: 해설에서는 출제 방향과 무관하게 항상 노출(이미 정답 공개됨). 대표 읽기 발음 재생.

### 4-4. 진행
- **"다음 문제"** 버튼으로 사용자가 직접 진행(자동 넘김 X).
- 10번째 문제 해설에서는 버튼이 **"완료"** → 서버에서 새 세트 조회 시도 → 쿨다운 중이면 **쿨다운 카운트다운 화면**(잠금화면 내 전환, 홈 이동 없음).
  - 오프라인 상태로 10번째 완료 시: 서버 조회 실패 → MMKV 캐시 사용. 캐시 커서가 만료되면 noContent("온라인이 되면 새 문제를 받아올게요") 표시. 쿨다운 카운트다운 없음(온라인 전환 후 sync 시 시작).

## 5. 오프라인 동작 (UX 관점)

- **감지 시 상단에 얇은 배너**: "오프라인 모드 · 푼 문제는 기록되지만 상자는 받을 수 없어요".
- 문제 풀이·해설·북마크·음성(기기 오프라인 음성 한정) **모두 동작**.
- 정답을 맞춰도 **상자 칩 대신** "오프라인이라 상자가 지급되지 않았어요" 안내.
- 다시 온라인이 되면: 푼 기록(SRS/통계)·북마크 토글을 **백그라운드 동기화**. 단, **상자는 소급 지급 안 함**(오프라인 풀이는 영구 무보상).
- 세트를 다 써서 캐시에 문제가 없고 오프라인이면: "온라인이 되면 새 문제를 받아올게요" 안내.

### 앱 종료 후 재진입 (세트 이어풀기)
- 세트 풀이 중 앱이 종료되거나 화면이 꺼져도, 로컬에 캐시된 세트(MMKV)와 진행 커서가 남아 있어 **재진입 시 이어서 풀기** 가능.
- **온라인으로 재진입**: 서버에서 현재 세트를 다시 조회해 로컬 캐시와 진행 상태를 서버 기준으로 맞춤. 오프라인 중 푼 답이 있으면 먼저 동기화(sync) 후 화면 복원.
- **오프라인으로 재진입**: MMKV에 저장된 세트와 커서로 이어서 진행. 동기화는 온라인 전환 시까지 대기.
- 캐시된 세트가 없고 오프라인이면: "온라인이 되면 새 문제를 받아올게요" 안내.

### 오프라인에서 세트 완료
- 오프라인 상태로 10번째 문제까지 모두 풀면, 로컬에서 세트 완료를 인식하고 `setDone` 화면 진입.
- 단, **서버에 `completed_at`이 기록되지 않아 쿨다운은 아직 시작 안 됨**. 온라인 전환 시 sync가 이루어지면 그때 `completed_at`이 기록되고 1시간 쿨다운이 시작됨.
- 이 상태에서는 "온라인이 되면 쿨다운이 시작돼요" 안내 표시. 쿨다운 카운트다운은 동기화 완료 후 시작.

## 6. 항목 타입별 해설 콘텐츠

| 학습 트랙 | 문제 대상 | 음성 | 부수/구성자 | 해설 표시 항목 |
|---|---|---|---|---|
| 한자 (`kanji`) | 한자 1자 | 음·훈독 | ✅ `components` + 획수 | 표기·음독·훈독·뜻·급수·구성자 |
| 한자단어 (`kanji_word`) | 한자 포함 단어 | 단어 읽기 | ✅ 단어 속 한자 구성자(가능 범위) | 표기·읽기·뜻·급수·(구성 한자) |
| 가나단어 (`kana_word`) | 가나 단어 | 단어 읽기 | ❌ | 표기·읽기·뜻·급수 |
| 가나 (`kana`) | 가나 1자 | 글자 발음 | ❌ | 글자·로마자·히라/가타 |

> ⚠️ 데이터 갭: **부수(radical) 전용 필드는 없음** — Kanji `components`(구성자)로 대체 표기. **한자단어→구성 한자 분해 데이터는 미보유** → MVP에서는 단어는 구성자 생략하거나, 단어 표기 내 한자를 런타임에 찾아 매핑하는 방식(구현 문서 참고).

## 7. 상태별 화면 정리

| 상태 | 화면 |
|---|---|
| 세트 풀이 중 | 문제 단계 ↔ 해설 패널 (3·4절) |
| 세트 완료(온라인) + 쿨다운 | "다음 세트까지 12:34" 카운트다운 (잠금화면 내 전환) |
| 세트 완료(오프라인) + sync 대기 | noContent("온라인이 되면 새 문제를 받아올게요") |
| 쿨다운 만료 | 카운트다운 자동 만료 → `loadSet()` 재호출 → 새 세트 로드 |
| 오프라인 + 캐시 있음 | 정상 풀이 + 오프라인 배너 (5절) |
| 오프라인 + 캐시 없음 | "온라인 시 새 문제를 받아올게요" |
| 출제 콘텐츠 부족 | 기존 NoContent 화면 유지 |

## 8. 변경/유지 요약

**바뀌는 것**
- 단일 문제 루프 → 10문제 세트 + 쿨다운.
- 1.2초 자동 넘김 → 해설 패널 + "다음 문제" 수동 진행.
- 음성 버튼, 북마크 버튼 신규.
- 오프라인 풀이 지원(무보상).
- 상자 캡: 일일 50 → (세트당 3) + (일일 50 상위 안전장치).

**유지되는 것**
- 정답 판정·상자 지급은 **서버 권위**(클라 is_correct 불신).
- 2×2 선택지, 잠금화면 다크 톤(인앱/잠금화면 공유 `LockQuizView`).
- SRS(SM-2)·QuizLog·Daily 집계 골격.
