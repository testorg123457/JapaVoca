# 로컬 실행 & 트러블슈팅 가이드 (Android)

새 머신에서 클론한 뒤 **빌드 → 에뮬레이터 실행 → 화면 확인**까지의 절차와, 실제로 겪었던 함정/해법을 모아둔 문서. (RN 0.86 / React 19 / New Architecture, 백엔드 Django + DRF + JWT, PostgreSQL)

---

## 1. 새 머신 셋업 체크리스트

클론 직후엔 아래가 없어서 "그냥 실행"은 안 된다. 버그가 아니라 정상 셋업 절차다.

1. **의존성 설치**
   - 모바일: `cd apps/mobile && npm install`
   - 서버: `cd apps/server && python -m venv .venv && source .venv/bin/activate && pip install -r requirements.txt`
2. **`.env` 재생성** ⚠️ — `.env`는 git에 올리지 않으므로 새 머신엔 없다. `.env.example`을 복사해서 채운다.
   - `apps/mobile/.env`: `API_BASE_URL=http://10.0.2.2:8000`, `GOOGLE_WEB_CLIENT_ID`(있으면), AdMob ID 등
   - `apps/server/.env`: `SECRET_KEY`, `DEBUG=True`, DB 접속정보, `ALLOWED_HOSTS=localhost,127.0.0.1` 등. **없으면 Django가 아예 안 뜬다.**
3. **PostgreSQL**: 로컬 DB면 설치 후 `python manage.py migrate`. 그리고 **단어 시드 데이터**가 있어야 퀴즈가 동작한다(퀴즈에 나오는 한자/단어는 DB에서 옴).
4. **Android SDK/NDK + AVD(에뮬레이터)** 가 그 머신에 설치돼 있어야 한다.
   - SDK 경로 예: `~/Library/Android/sdk`. PATH에 `platform-tools`, `emulator` 추가.
   - NDK는 RN이 핀하는 버전 + `cmake;3.22.1` 필요(CMake 4.x는 RN CMakeLists 깨짐).
   - `android/local.properties`에 `sdk.dir` 지정.

## 2. 실행 순서

세 개(에뮬레이터 / 백엔드 / Metro+앱)가 다 떠 있어야 한다.

```bash
# 1) 에뮬레이터 (cold boot 권장 — 아래 4번 네트워크 함정 예방)
emulator -avd Pixel_5 -no-snapshot -gpu swiftshader_indirect &

# 2) 백엔드 (0.0.0.0 으로 바인딩해야 에뮬레이터가 10.0.2.2 로 접근 가능)
cd apps/server && source .venv/bin/activate && python manage.py runserver 0.0.0.0:8000

# 3) Metro + 앱 (첫 빌드는 네이티브 컴파일로 10분+ 걸림 — 정상)
cd apps/mobile && npx react-native run-android
```

- 에뮬레이터에서 호스트(내 PC)는 **`10.0.2.2`** 로 접근한다. Metro=`10.0.2.2:8081`, 백엔드=`10.0.2.2:8000`.
- **첫 Gradle 빌드는 머신마다 한 번씩 오래 걸린다**(reanimated 등 C++ 컴파일). 버그 아님.

### DEBUG 전용 dev 로그인
구글 OAuth(`GOOGLE_WEB_CLIENT_ID`) 설정 없이 로그인 이후 화면을 보려면, 로그인 화면 하단의 **"개발용 로그인 (DEBUG)"** 버튼을 누른다. 고정 테스트 유저로 JWT를 발급한다. (`POST /api/auth/dev-login/`, `DEBUG=False`면 URL 미등록·404)

## 3. 코드에 박혀 해결된 이슈 (재발 안 함, 기록용)

- **debug 빌드인데 `loadJSBundleFromAssets`로 크래시** → `apps/mobile/android/app/src/main/java/com/japavoca/app/MainApplication.kt`에서 `getDefaultReactHost(... , useDevSupport = BuildConfig.DEBUG)` 명시로 해결. 원인: 2-인자 `getDefaultReactHost`의 `useDevSupport` 기본값이 `ReactBuildConfig.DEBUG`(=**react-android 릴리스 AAR**의 BuildConfig.DEBUG=false)라, debug 빌드도 Metro를 안 거치고 없는 assets 번들을 찾다 크래시했다. 반드시 **앱 자신의** `BuildConfig.DEBUG`를 넘겨야 한다.
- **앱→서버 요청이 400 Bad Request** → `apps/server/config/settings.py`에서 `DEBUG`일 때 `ALLOWED_HOSTS`에 `10.0.2.2` 자동 추가로 해결. (에뮬레이터가 보내는 `Host: 10.0.2.2` 헤더가 거부되던 문제)

---

## 4. 환경 이슈 (머신/에뮬레이터 상태 따라 또 날 수 있음 — 해법)

- **앱 uid만 네트워크 전체 불통** (shell uid는 정상, Metro `10.0.2.2`·외부 둘 다 타임아웃, `dumpsys netpolicy`엔 아무것도 없음).
  - 원인: 스냅샷에서 복원된 에뮬레이터의 stale per-uid 네트워크/방화벽 상태.
  - 해법: **cold boot**. 모든 qemu kill → `rm -rf ~/.android/avd/Pixel_5.avd/*.lock` → `emulator -avd Pixel_5 -no-snapshot -gpu swiftshader_indirect`.
- **`adb reverse`(localhost:8081)가 이 에뮬레이터에선 동작 안 함.** 대신 RN 기본값인 **`10.0.2.2:8081`이 정상 동작**하니 `debug_http_host`를 localhost로 덮어쓰지 말 것(오히려 망가짐).
- **`screencap` 스크린샷이 새까맣게 나옴** (HWUI 101010-2 에러) → 에뮬레이터를 `-gpu swiftshader_indirect`로 실행. swiftshader에선 System UI가 가끔 ANR 나지만 앱엔 무해.
- Metro 없이 UI만 빨리 확인: `npx react-native run-android --mode release --active-arch-only` (릴리스는 JS 번들을 내장 → Metro/reverse 불필요).

## 5. 기타 빌드 함정 (초기 셋업 때 겪음)

- **누락되기 쉬운 peer deps**: `react-native-worklets`(Reanimated 4 peer; babel 플러그인 `react-native-worklets/plugin`은 **맨 마지막**), `react-native-nitro-modules`(react-native-mmkv v4 / NitroMmkv peer — 없으면 `:react-native-mmkv` 구성 실패).
- **AdMob**: `react-native-google-mobile-ads`만 설치하면 `MobileAdsInitProvider`가 부팅 시 크래시. `AndroidManifest.xml`에 `<meta-data com.google.android.gms.ads.APPLICATION_ID>` 필요(현재 Google 테스트 ID `ca-app-pub-3940256099942544~3347511713` + `tools:replace="android:value"`). 실서비스 땐 `.env`의 `ADMOB_APP_ID`로 교체.
- **buildToolsVersion**은 설치된 버전(36.1.0)으로 맞춤(36.0.0 미설치 시 에러).
- 디자인 시스템: gluestack-ui는 **제거됨**(v1이 react-dom을 끌어와 RN 번들 깨짐). UI는 `src/theme/tokens.ts` + NativeWind, 공용 컴포넌트는 `src/components/`.
