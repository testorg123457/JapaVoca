> 상태: **검토 대기** — 승인 전까지 코드 수정하지 않는다.
> 작성 2026-06-25. 대상: `apps/mobile`. 트리거: 실기기에서 (1)지갑·설정 탭이 안 눌림 (2)하단 광고 짤림 (3)상단 캐시 카드 깨짐 + 참고로 받은 알라미(Alarmy) 다크 톤 스크린샷 3장.

## 1. 문제 & 근본 원인 (코드 확인 완료)

### 버그 A — 지갑/설정 탭이 "너무 아래라 안 눌림"
- 위치: `src/navigation/BottomTabNavigator.tsx`
- 원인: `tabBarStyle`에 `height: 62`, `paddingBottom: 8`을 **고정**해서, 제스처 네비게이션 바(홈 인디케이터)가 있는 기기에서 탭바가 시스템 제스처 영역에 깔린다. React Navigation이 자동 적용하는 하단 safe-area inset이 고정 height에 덮여 무효화됨 → 탭 하단 절반이 제스처 영역과 겹쳐 탭이 안 먹는 느낌.
- 영향: 출석 체크(화면 중앙)는 멀쩡히 눌리는데 하단 탭만 안 눌리는 증상과 정확히 일치.

### 버그 B — 하단 배너 광고가 짤림
- 위치: `src/screens/main/HomeScreen.tsx` (288–299줄). `SafeAreaView edges={['top']}` (하단 inset 없음) 안에서 배너가 맨 아래 `View`로 탭바 위에 바로 붙어 있음.
- 원인: ① 하단 safe-area 미적용 ② 광고가 **홈 화면에만** 존재(탭바 위) → 탭바·홈인디케이터와 공간을 다퉈 잘림. 지갑/설정엔 광고가 아예 없어 화면마다 구조가 들쭉날쭉.

### 버그 C — 상단 캐시 카드(그라데이션) 깨짐
- 위치: `src/components/Gradient.tsx`. `<Svg width="100%" height="100%">` + `<Rect width="100%" height="100%">`.
- 원인: react-native-svg의 퍼센트 사이즈가 **New Architecture(Fabric)에서 첫 레이아웃 측정과 어긋나** 직사각형이 카드 전체 폭을 못 채움 → 우측에 색이 끊긴 seam(깨진 띠)이 보임.

### 참고: 맨 아래 노란 `!` 토스트
- React Native **LogBox(dev 경고 오버레이)**. DEBUG에서만 뜨고 release엔 없음. 광고 아님. 별도 수정 불필요(릴리스에서 사라짐).

## 2. 수정안

### A. 하단 탭바 — safe-area inset 적용 (`BottomTabNavigator.tsx`)
- `useSafeAreaInsets()`의 `bottom`을 반영: `height: 62 + insets.bottom`, `paddingBottom: 8 + insets.bottom`.
- 결과: 탭 아이콘/라벨이 제스처바 위로 올라와 탭 영역이 온전히 눌린다.

### B. 배너 광고 — 전역(탭바 아래) 1곳으로 이동 ⭐ (알라미 패턴)
- 현재: 홈에만, 탭바 *위*, safe-area 없음.
- 변경: 배너를 **탭 네비게이터를 감싸는 래퍼**(예: `BottomTabNavigator`를 감싼 View 또는 신규 `MainTabsWithAd`)로 빼서 **탭바 아래·화면 맨 하단**에 고정. 하단 safe-area 패딩 적용.
  - 알라미처럼 모든 메인 화면(홈/지갑/설정)에서 동일하게 한 줄 광고가 깔리고, 절대 잘리지 않음.
  - 홈 화면(`HomeScreen`)의 개별 배너 코드(288–299줄)는 제거.
  - 로드 실패 시 영역 자체 숨김(현 `adFailed` 로직 유지).
- 순서(위→아래): 화면 콘텐츠 → 탭바 → 배너광고 → (홈 인디케이터 safe-area).

### C. 그라데이션 — 퍼센트 대신 측정된 픽셀 사용 (`Gradient.tsx`)
- `onLayout`으로 부모 크기를 px로 측정해 `<Svg width={w} height={h}>` + `<Rect width={w} height={h}>`에 숫자값을 넘긴다(측정 전 0이면 미렌더). 퍼센트 의존 제거 → seam 사라짐.
- (대안) `react-native-linear-gradient` 네이티브 라이브러리로 교체. 더 견고하지만 **네이티브 의존 추가 = 재빌드 필요**. 1차는 onLayout 방식 권장(JS만, 재빌드 불필요).

## 3. "알라미 느낌" 반영 범위 — 결정 필요 ⚠️

참고 스크린샷 3장은 모두 **알라미(다크 테마)**다. 어디까지 맞출지 결정이 필요하다. 현재 앱은 **민트-라이트**(배민 톤)이고, 다크 모드 토큰/Provider는 이미 구현돼 있어 시스템 다크 모드를 따라간다.

**무드(테마) 방향 — 셋 중 택1:**
- **옵션 1 (권장): 라이트 유지 + 마감만 알라미급으로.** 버그 A·B·C 고치고, 하단 탭+광고 정돈, 카드/섹션 여백·정렬 다듬기. 캐시 리워드 앱은 보통 라이트(캐시워크 계열)라 정체성에 맞음. 작업량 작음.
- **옵션 2: 다크 테마로 전환.** 토큰의 dark 세트를 기본으로 강제(시스템 무관) + 알라미식 카드/구분. 토큰 인프라는 있으나 전 화면 대비·가독성 재검수 필요. 작업량 중간.
- **옵션 3: 라이트/다크 토글 제공.** 설정에서 사용자가 선택. 작업량 큼(MVP엔 과함).

> 이 문서의 §2(A·B·C)는 **어느 옵션이든 공통으로 필요한 버그 수정**이다. §3은 그 위에 얹는 비주얼 방향.
