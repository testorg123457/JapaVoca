JapaVoca — "밀어서 잠금해제" 제스처 수정 플랜

> 증상: 퀴즈 화면(`LockQuizScreen`)에서 오른쪽으로 밀면 **손가락을 따라 페이지가 밀리지 않고 그냥 사라짐**.
> 손가락 추적(드래그 따라 이동)이 핵심 요구사항. 작업 규칙: [`CLAUDE.md`](../CLAUDE.md).

---

## 1. 현재 구조

- 진입: 홈 "퀴즈 시작" → `navigation.navigate('LockQuiz')`. `MainStack`에서 **`presentation: 'fullScreenModal'`** 로 등록.
- 제스처: `LockQuizView` 안에서 `Gesture.Pan()` → 공유값 `dragX` 갱신 → `Animated.View`에 `translateX`.
  ```
  Gesture.Pan()
    .activeOffsetX(20).failOffsetY([-14,14])
    .onUpdate(e => dragX.value = max(0, e.translationX))   // 손가락 추적
    .onEnd(e => 임계 넘으면 withTiming(600)+onUnlock, 아니면 spring(0))
  ```
- 환경: RN New Arch, **react-native-gesture-handler 3.0.2**, **reanimated 4.4.1**, **react-native-worklets 0.9.2**.

---

## 2. 원인 가설 (우선순위)

이번까지 시도(메모이즈·useWindowDimensions·속도/지연·`GestureHandlerRootView` 래핑)로 **해결 안 됨** → 단일 원인이 아닐 가능성.

1. **모달 컨테이너 분리** — `fullScreenModal`은 react-native-screens가 **별도 네이티브 윈도우**로 띄워, App 최상단 `GestureHandlerRootView` 바깥일 수 있음 → 모달 안 RNGH 제스처가 인식 불안정. (래핑 시도했으나 미해결 → 단독 원인은 아님)
2. **자식 Pressable의 터치 가로채기** — 화면 대부분이 `PressableScale`(RN Pressable: 선택지 4개·버튼들). Pan이 `activeOffsetX`로 활성화되기 전 Pressable이 responder를 잡으면 드래그가 Pan으로 안 넘어옴. "사라짐"이 사실은 **선택지 탭 → 채점 → 다음문제 자동전환**일 수 있음.
3. **`failOffsetY([-14,14])` 조기 실패** — 살짝 비스듬한 스와이프(세로 14px 초과)면 Pan이 **취소(fail)** → 드래그 안 됨.
4. **gesture 콜백 워클릿화/공유값 브리지** — RNGH3+reanimated4+worklets에서 gesture 콜백이 UI스레드 워클릿으로 안 도는 경우 `dragX` 갱신이 화면에 반영 안 됨. (단, 다른 reanimated 애니메이션은 정상 → 가능성 낮음)

## 3. 수정 전략

### 3-A. 핵심 수정 (이번에 적용)
1. **프레젠테이션 변경**: `LockQuiz`를 `fullScreenModal` → **일반 카드 화면**(default, 필요시 `animation:'fade'`)으로. → 가설 1(모달 컨테이너) 제거. 화면은 그대로 전체를 덮음.
2. **`GestureHandlerRootView`로 `LockQuizView` 감싸기** 유지(belt-and-suspenders, 카드에서도 무해).
3. **제스처 견고화**:
   - `failOffsetY` **제거**(가설 3) — 비스듬해도 안 끊김.
   - `activeOffsetX(15)` 로 오른쪽 가로 드래그에만 활성(탭/세로는 통과).
   - 콜백 **명시적 `'worklet'`**(가설 4) — UI스레드 보장.
   - 손가락 추적: `dragX = clamp(translationX, 0, width)`.
   - 놓을 때: 임계(폭 25%)·속도(>800) 넘으면 **끝까지 슬라이드 후 `onUnlock`(완료 콜백)**, 아니면 spring 복귀.
4. **Pressable 경합 대비(가설 2)**: 선택지 등 `PressableScale`이 Pan을 막지 않도록, 필요 시 Pan에 `simultaneousWithExternalGesture` 또는 선택지 영역을 제외한 **상/하단 안전영역에서도 드래그 가능**하게 확인. 1차로는 whole-screen Pan 유지하되 안 되면 5단계로.

### 3-B. 검증용 디버그 (확실히 하기 위해)
- 임시 **드래그 값 표시**(`dragX`를 화면 상단에 숫자로) 또는 별도 **디버그 화면**(Pressable 없는 빈 화면 + 드래그 박스)으로 "RNGH Pan + reanimated가 이 앱에서 동작하는가"를 **격리 확인**.
  - 디버그 박스가 따라오면 → 범인은 모달/Pressable. 안 따라오면 → 워클릿/브리지.
- 확인 후 디버그 코드는 제거.

### 3-C. 폴백 (3-A로도 안 되면, 택1)
- **위로 밀어서 잠금해제**(상하 스와이프) — 시스템 가로 뒤로가기·가로 Pressable과 충돌 없음, 실제 잠금화면 관용 패턴.
- **하단 슬라이더 핸들** 영역만 드래그(자식 Pressable 경합 원천 차단).
- 제스처 포기 → **하단 "잠금해제" 버튼**으로 단순화.

---

## 4. 작업 순서
1. 디버그 격리 테스트(3-B) 넣고 1회 확인 → 원인 좁히기.
2. `MainStack`: `LockQuiz` 프레젠테이션 변경(3-A-1).
3. `LockQuizView`: `GestureHandlerRootView` 래핑 + 제스처 견고화(3-A-2,3).
4. 실기기 확인(손가락 추적 / 복귀 / 슬라이드아웃).
5. 안 되면 폴백(3-C) 적용. 디버그 코드 제거.

## 5. 검증 기준
- 천천히 밀면 **페이지가 손가락 따라 이동**.
- 충분히 밀고 놓으면 **끝까지 슬라이드 후 닫힘**, 부족하면 **제자리 복귀**.
- 선택지 탭(채점)·세로 스크롤과 **충돌 없음**.
- 인앱 진입(홈) + 잠금화면(LockApp) **둘 다** 동작.
